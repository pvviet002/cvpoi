Kiểm tra tích số trên đường đi

Làm sao để biết một số nguyên nhân với chuỗi số thực có sinh ra số nguyên không, mà không bao giờ bị sai số thập phân?

MINI-LAB: Phân tích phép nhân số thực

Phân tích: 21 · 7
×
Phân số: 25 / 100 = 1 / 22
=
Kết quả:
3.5 (Không)

Bản chất: Để kết quả là số nguyên, lượng thừa số 2 và 5 ở mẫu số của W phải bị triệt tiêu hết bởi thừa số 2 và 5 trong X.

1.Tóm tắt đề bài

  • Cho một cây ~ n đỉnh, mỗi đỉnh i có trọng số nguyên ai.
  • Mỗi cạnh có trọng số thực w ≥ 0 (tối đa 4 chữ số phần thập phân).
  • Cho q truy vấn (x, y). Hỏi tích: ax · (tích các w trên đường đi từ x đến y) có phải là số nguyên không?
  • Ràng buộc: n, q ≤ 2 · 105; ai ≤ 109; w ≤ 104.

2.Phân tích ban đầu

Cách tự nhiên nhất là DFS từ x đến y, nhân dồn các số thực w lại rồi kiểm tra tích == floor(tích). Tại sao cách này thất bại hoàn toàn?

  • Sai số thập phân (Precision Loss): Máy tính lưu số thực dưới dạng IEEE 754. Ví dụ 0.1 + 0.2 ≠ 0.3. Việc nhân liên tục hàng chục số thực sẽ dẫn đến sai số rất lớn, hàm kiểm tra số nguyên sẽ chạy sai bét.
  • Tràn số (Overflow/Underflow): Trọng số w có thể nhỏ (0.0001) hoặc lớn (10000). Nhân liên tục trên đường đi dài 105 cạnh sẽ khiến số tiến về 0 tuyệt đối (underflow) hoặc vượt giới hạn double (overflow).
  • Chậm: Làm thủ công từng truy vấn tốn O(n), q truy vấn tốn O(n · q) ≈ 4 · 1010 thao tác → Quá thời gian quy định (TLE).

Chúng ta cần chuyển bài toán từ "nhân số thực" sang "đếm số nguyên", và từ "chạy lại đường đi" sang "trang bị cấu trúc dữ liệu trên cây".

3.Mô-đun 1: Giải phẫu một số thực

(a) Ta muốn gì? Xóa sổ hoàn toàn kiểu double, đưa mọi thứ về số nguyên để tính toán chính xác tuyệt đối.

(b) Quan sát then chốt: Mọi số thực w có tối đa d chữ số thập phân đều có thể viết dưới dạng phân số: w = V / 10d (với V là phần số nguyên khi bỏ dấu chấm). Ví dụ: 0.25 = 25 / 100.

(c) Ý tưởng: Mẫu số luôn là lũy thừa của 10. Mà 10 = 2 · 5, nên mẫu số chỉ chứa hai thừa số nguyên tố là 2 và 5. Để tích ax · (V1 / 10d1) · (V2 / 10d2) ... là số nguyên, thì tổng số lượng thừa số 2 (và 5) trên Tử Số phải lớn hơn hoặc bằng Mẫu Số. Những thừa số nguyên tố khác (3, 7, 11...) trên Tử Số không bao giờ bị Mẫu Số triệt tiêu, nên ta không cần quan tâm đến chúng!

(d) Vì sao đúng? Gọi cnt2(Z) là số lần chia hết cho 2 của Z. Với một cạnh có w = V / 10d, "lợi nhuận ròng" về thừa số 2 của nó là: C2 = cnt2(V) - d (có thể âm). Tương tự C5 = cnt5(V) - d. Ta chỉ cần cộng dồn C2 và C5. Nếu tổng cuối cùng ≥ 0 thì là số nguyên!

Minh họa biến đổi C2 và C5

Input w
0.20
→
Phân số (V, d)
20 / 102
→
Phân tích Tử
22 · 51
→
Lợi nhuận ròng
C2 = 2 - 2 = 0
C5 = 1 - 2 = -1

Nhận xét: Đi qua cạnh 0.20 sẽ "nuốt" mất 1 thừa số 5 của ax ban đầu. Nếu ax không có thừa số 5 nào bù đắp, kết quả sẽ rớt thành phân số.

Kiểm tra nhanh: Đỉnh xuất phát x có ax = 24. Đi qua duy nhất một cạnh có trọng số w = 0.5. Tích có phải là số nguyên không?
A. Có, vì 24 · 0.5 = 12 là số nguyên.
B. Không, vì w là số thập phân.
C. Không, vì ax = 24 không có thừa số 5 nào.

4.Mô-đun 2: Rút gọn truy vấn trên cây

(a) Ta muốn gì? Tính nhanh tổng C2 và tổng C5 trên một đường đi bất kỳ (x, y) trong thời gian O(log n).

(b) Khó khăn: Đường đi trên cây có thể uốn lượn đi lên rồi đi xuống. Ta không thể for-loop từng cạnh.

(c) Ý tưởng: Dùng Mảng cộng dồn trên cây (Prefix Sum). Cố định đỉnh 1 làm gốc. Gọi dist2[u] là tổng C2 từ gốc 1 đến đỉnh u. Tổng C2 trên đường đi (x, y) chính là tổng quãng đường từ x lên Tổ tiên chung gần nhất (LCA) cộng với quãng đường từ y lên LCA:
Sum_C2 = dist2[x] + dist2[y] - 2 · dist2[lca(x, y)].

(d) Ngoại lệ CHẾT NGƯỜI: Chuyện gì xảy ra nếu w = 0? Nếu trên đường đi có số 0, thì tích luôn bằng 0 (là số nguyên) bất chấp đếm 2 hay đếm 5! Ta cần đếm số lượng cạnh 0 bằng mảng cnt0 tương tự.

Tại sao khi gặp w = 0 ta phải tách ra đếm riêng (cnt0) mà không gộp chung vào C2 và C5?
A. Vì 0 không có phân tích ra thừa số nguyên tố, nếu cố tính cnt2(0) chương trình sẽ bị lặp vô hạn (chia cho 2 mãi mãi).
B. Vì 0 làm tràn biến đếm.
C. Vì đề bài cấm trọng số w = 0.

Thuật toán từng bước

  1. Đọc và xử lý trọng số mỗi cạnh:
    • Đọc w dưới dạng chuỗi (string) để tránh sai số.
    • Tìm vị trí dấu chấm. Nếu không có, d = 0, V = stoi(w).
    • Nếu có dấu chấm, gọi d là số ký tự sau dấu chấm. Xóa dấu chấm đi và chuyển chuỗi còn lại thành số nguyên V.
    • Tính C2 = cnt2(V) - d và C5 = cnt5(V) - d. (Trong đó cnt2(x) là số lần chia hết cho 2 của x).
    • Nếu V == 0, đánh dấu is0 = 1, ngược lại is0 = 0.
  2. Duyệt cây (DFS) và chuẩn bị Binary Lifting:
    • Từ gốc (đỉnh 1), gọi hàm DFS đi qua các đỉnh.
    • Với đỉnh u đến từ parent, cập nhật: dist2[u] = dist2[parent] + C2, tương tự cho dist5[u] và cnt0[u].
    • Khởi tạo mảng up[u][j] lưu tổ tiên bậc 2j của u để tìm LCA nhanh.
  3. Trả lời truy vấn (x, y):
    • Nếu a_x == 0, in Yes ngay lập tức.
    • Tìm lca = get_LCA(x, y).
    • Tính số cạnh 0 trên đường đi: Z = cnt0[x] + cnt0[y] - 2 · cnt0[lca]. Nếu Z > 0, in Yes.
    • Tính tổng C2_sum = dist2[x] + dist2[y] - 2 · dist2[lca]. Tương tự cho C5_sum.
    • Kiểm tra: Nếu cnt2(a_x) + C2_sum ≥ 0 VÀ cnt5(a_x) + C5_sum ≥ 0, in Yes. Ngược lại in No.

5.Cài đặt (C++)

#include <iostream>
#include <vector>
#include <string>
using namespace std;
const int MAXN = 200005;
const int LOG = 20;
struct Edge {
int v, c2, c5, is0;
};
vector<Edge> adj[MAXN];
int a[MAXN];
int up[MAXN][LOG];
int depth[MAXN];
long long dist2[MAXN], dist5[MAXN], cnt0[MAXN];
int get_cnt(int val, int prime) {
if (val == 0) return 0; // Tránh lặp vô hạn
int res = 0;
while (val % prime == 0) {
res++;
val /= prime;
}
return res;
}
void dfs(int u, int p) {
up[u][0] = p;
for (int i = 1; i < LOG; i++) {
up[u][i] = up[up[u][i-1]][i-1];
}
for (Edge e : adj[u]) {
int v = e.v;
if (v == p) continue;
depth[v] = depth[u] + 1;
dist2[v] = dist2[u] + e.c2;
dist5[v] = dist5[u] + e.c5;
cnt0[v] = cnt0[u] + e.is0;
dfs(v, u);
}
}
int get_lca(int u, int v) {
if (depth[u] < depth[v]) swap(u, v);
for (int i = LOG - 1; i >= 0; i--) {
if (depth[u] - (1 << i) >= depth[v])
u = up[u][i];
}
if (u == v) return u;
for (int i = LOG - 1; i >= 0; i--) {
if (up[u][i] != up[v][i]) {
u = up[u][i];
v = up[v][i];
}
}
return up[u][0];
}
int main() {
ios_base::sync_with_stdio(false); cin.tie(NULL);
int n, q;
if (!(cin >> n >> q)) return 0;
for (int i = 1; i <= n; i++) cin >> a[i];
for (int i = 1; i < n; i++) {
int u, v; string w_str;
cin >> u >> v >> w_str;
// Xử lý chuỗi số thực
int dot = w_str.find('.');
int d = 0, V = 0;
if (dot == string::npos) {
V = stoi(w_str);
} else {
d = w_str.length() - dot - 1;
w_str.erase(dot, 1);
V = stoi(w_str);
}
int is0 = (V == 0 ? 1 : 0);
int c2 = get_cnt(V, 2) - d;
int c5 = get_cnt(V, 5) - d;
adj[u].push_back({v, c2, c5, is0});
adj[v].push_back({u, c2, c5, is0});
}
dfs(1, 1);
while (q--) {
int x, y; cin >> x >> y;
if (a[x] == 0) { cout << "Yes\n"; continue; }
int lca = get_lca(x, y);
long long Z = cnt0[x] + cnt0[y] - 2 * cnt0[lca];
if (Z > 0) { cout << "Yes\n"; continue; }
long long sum2 = dist2[x] + dist2[y] - 2 * dist2[lca];
long long sum5 = dist5[x] + dist5[y] - 2 * dist5[lca];
if (get_cnt(a[x], 2) + sum2 >= 0 && get_cnt(a[x], 5) + sum5 >= 0) {
cout << "Yes\n";
} else {
cout << "No\n";
}
}
return 0;
}

6.Độ phức tạp & Cạm bẫy

Thời gian: DFS duyệt qua n đỉnh tốn O(n log n) do khởi tạo bảng Binary Lifting. Mỗi truy vấn tìm LCA mất O(log n), tổng cho q truy vấn là O(q log n). Tổng thời gian O((n + q) log n). Với N, Q ≤ 2 · 105, số phép tính khoảng 7 · 106, chạy dư sức trong 1 giây.

Không gian: Mảng up tốn O(n log n). Các mảng dist tốn O(n). Tổng cộng O(n log n) bộ nhớ, rất an toàn.

✗ Lỗi sai thường gặp
  • Dùng double hoặc long double để nhân trực tiếp → Chắc chắn sai số sau vài chục phép nhân.
  • Gọi cnt2(0) trong vòng lặp while (val % 2 == 0) → Treo máy (Time Limit Exceeded) vì 0 chia hết cho mọi số.
  • Chuyển string sang float rồi mới nhân 10d → Vẫn bị dính sai số của hàm parse float.
✓ Cách xử lý đúng
  • Quy đổi mọi số về cặp giá trị (số lượng thừa số 2, số lượng thừa số 5).
  • Bắt điều kiện rẽ nhánh sớm: Nếu V == 0 thì không gọi cnt2, gán luôn is0 = 1.
  • Thao tác chuỗi thuần túy: Tách thủ công dấu chấm . rồi dùng stoi() là chuẩn xác 100%.
