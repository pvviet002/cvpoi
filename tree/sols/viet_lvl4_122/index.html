Graph/Tree Mo's Algorithm Number Theory

Đếm số ước nguyên tố trên đường đi

Thay vì duyệt cây tốn kém cho mỗi truy vấn, ta biến đổi cấu trúc cây thành mảng 1D bằng Dãy Euler (Euler Tour), sau đó dùng Thuật toán Mo trượt trên mảng để đếm số lượng thừa số nguyên tố phân biệt với tốc độ chớp nhoáng.

Tóm tắt đề bài

Cho một cây gồm n đỉnh. Đỉnh i có trọng số ai. Giả sử mỗi ước nguyên tố của ai là một "công ty".

Có q truy vấn dạng (u, v). Với mỗi truy vấn, hãy đếm số lượng công ty (ước nguyên tố) phân biệt trên đường đi từ đỉnh u đến đỉnh v.

Ràng buộc:

  • n, q ≤ 5 × 104
  • ai ≤ 105

Ví dụ ngẫu nhiên:

Cây 3 đỉnh: a = [7, 2, 1]
Cạnh: (1, 2), (1, 3)
Truy vấn: (2, 3)
Đường đi 2 → 1 → 3 có các đỉnh mang giá trị [2, 7, 1].
Thừa số nguyên tố: của 2 là {2}, của 7 là {7}, của 1 là rỗng.
Tổng hợp lại có tập {2, 7} → Đáp án: 2.

Phân tích ban đầu

Ta muốn gì? Cần tìm số nguyên tố phân biệt xuất hiện trong các số ai trên một đường đi bất kỳ.

Cách ngây thơ: Với mỗi truy vấn (u, v), ta dùng DFS/BFS duyệt tìm đường đi (mất O(n) đỉnh). Tại mỗi đỉnh, ta phân tích ai ra thừa số nguyên tố và bỏ vào tập hợp (Set). Cuối cùng đếm kích thước tập hợp.

Vì sao thất bại? Có q = 50000 truy vấn. Nếu đường đi dài 50000 đỉnh, thời gian cho toàn bộ sẽ là O(q × n × (thời_gian_phân_tích)) ≈ 2.5 × 109 phép toán. Chắc chắn quá thời gian (TLE)!

Hướng giải quyết: Khi bài toán yêu cầu trả lời nhiều truy vấn trên một cấu trúc tĩnh (cây không bị thay đổi), và việc gộp/bỏ một phần tử vào kết quả rất dễ, ta phải nghĩ ngay đến Thuật toán Mo kết hợp với kỹ thuật Trải phẳng cây.

Bước 1: Tiền xử lý thừa số nguyên tố

Ta muốn gì? Tách mọi ai thành danh sách các ước nguyên tố một cách nhanh nhất, và không lưu trùng lặp tại một đỉnh (vd 12 = 2 × 2 × 3 ta chỉ quan tâm nó mang công ty số 2 và 3).

Quan sát: Giá trị ai ≤ 105 rất nhỏ. Ta có thể dùng Sàng Eratosthenes để tìm Ước nguyên tố nhỏ nhất (Smallest Prime Factor - SPF) của mọi số từ 1 đến 105 trong thời gian O(M \log \log M) với M = 105.

Cách làm: Tại mỗi đỉnh, dùng mảng SPF chia liên tục ai để lấy các nguyên tố. Mỗi đỉnh sẽ được thay thế bằng một mảng nhỏ chứa các số nguyên tố (tối đa khoảng 7 số vì 2 × 3 × 5 × 7 × 11 × 13 × 17 > 10^{5}).

Bước 2: Trải phẳng cây (Euler Tour)

Ta muốn gì? Thuật toán Mo chỉ hoạt động trên mảng 1 chiều. Nhưng dữ liệu của ta là một cái cây. Ta cần biến các đường đi ngoằn ngoèo trên cây thành các đoạn liên tiếp [L, R] trên mảng.

Ý tưởng (Euler Tour): Ta duyệt DFS cây. Mỗi đỉnh sẽ được ghi vào mảng đúng 2 lần: lần đầu khi đi vào đỉnh (ST) và lần cuối khi thoát khỏi đỉnh đó (EN). Mảng kết quả có kích thước 2n.

Sự kỳ diệu: Giả sử cần truy vấn đường đi từ u đến v (quy ước STu ≤ STv).
- Nếu u là tổ tiên của v: Đường đi tương ứng với đoạn [STu, STv].
- Nếu u không phải tổ tiên của v: Đường đi tương ứng với đoạn [ENu, STv]. Quan trọng: đoạn này chưa chứa LCA(u, v) nên ta phải gộp thêm đỉnh LCA vào kết quả.

Đặc biệt: Bất kỳ đỉnh nào xuất hiện chính xác 2 lần trong đoạn truy vấn có nghĩa là ta đã đi vào nhanh nhánh đó và quay ra, nó KHÔNG nằm trên đường đi trực tiếp! Ta sẽ triệt tiêu (bỏ qua) các đỉnh xuất hiện 2 lần.

Bước 3: Thuật toán Mo trên mảng trải phẳng

Ta muốn gì? Giờ ta có mảng E kích thước 2n. Các truy vấn được đổi thành [L, R]. Ta cần đếm số nguyên tố phân biệt.

Cách làm:
1. Sắp xếp các truy vấn [L, R] theo khối (block size B ≈ N/\sqrt{Q}) để tối ưu việc di chuyển con trỏ.
2. Dùng 2 con trỏ l và r trượt dọc mảng E.
3. Quy tắc Toggle (Bật/Tắt): Khi con trỏ mở rộng/thu hẹp gặp đỉnh u, ta đổi trạng thái của u. Nếu u từ "tắt" thành "bật", ta xét các ước nguyên tố của nó: thêm vào mảng đếm tần suất. Nếu một nguyên tố chuyển từ 0 lên 1, tổng số lượng công ty ans tăng lên. Nếu u từ "bật" thành "tắt", ta trừ tần suất đi.

Nhờ quy tắc Toggle, những đỉnh nằm hoàn toàn trong đoạn [L, R] nhưng xuất hiện 2 lần sẽ bị "bật" rồi lại "tắt", cuối cùng không ảnh hưởng đến kết quả! Đúng chuẩn xác yêu cầu đường đi.

Cài đặt

Toàn bộ lời giải bằng C++. Chú ý cách cài đặt hàm toggle và cách thêm tạm LCA vào kết quả.

// Cấu trúc lệnh: O(N * sqrt(Q))
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 50005;
const int MAXA = 100005;
int n, q, a[MAXN], B;
vector<int> adj[MAXN];
vector<int> primes_of[MAXN]; // Mảng chứa các ước nguyên tố phân biệt của từng đỉnh
// Sàng nguyên tố (SPF)
int spf[MAXA];
void sieve() {
for (int i = 1; i < MAXA; i++) spf[i] = i;
for (int i = 2; i * i < MAXA; i++) {
if (spf[i] == i) {
for (int j = i * i; j < MAXA; j += i)
if (spf[j] == j) spf[j] = i;
}
}
}
// Euler Tour & LCA Variables
int st[MAXN], en[MAXN], timer = 0, E[MAXN * 2];
int up[MAXN][17], depth[MAXN];
void dfs(int u, int p = 0) {
st[u] = ++timer;
E[timer] = u;
up[u][0] = p;
for (int i = 1; i < 17; i++) up[u][i] = up[up[u][i-1]][i-1];
for (int v : adj[u]) {
if (v != p) {
depth[v] = depth[u] + 1;
dfs(v, u);
}
}
en[u] = ++timer;
E[timer] = u; // Ghi u lần 2 khi thoát
}
int getLCA(int u, int v) {
if (depth[u] < depth[v]) swap(u, v);
for (int i = 16; i >= 0; i--)
if (depth[u] - (1 << i) >= depth[v]) u = up[u][i];
if (u == v) return u;
for (int i = 16; i >= 0; i--)
if (up[u][i] != up[v][i]) u = up[u][i], v = up[v][i];
return up[u][0];
}
// Cấu trúc Query cho Mo
struct Query {
int l, r, lca, id;
bool operator<(const Query &other) const {
if (l / B != other.l / B) return l / B < other.l / B;
// Tối ưu chạy dích dắc của r
return ((l / B) & 1) ? r < other.r : r > other.r;
}
};
int active[MAXN], cnt[MAXA], cur_ans = 0;
// Hàm Toggle thần thánh
void toggle(int u) {
if (active[u]) { // Nếu đang Bật → Tắt đi
for (int p : primes_of[u]) {
cnt[p]--;
if (cnt[p] == 0) cur_ans--;
}
} else { // Nếu đang Tắt → Bật lên
for (int p : primes_of[u]) {
if (cnt[p] == 0) cur_ans++;
cnt[p]++;
}
}
active[u] ^= 1; // Đảo trạng thái
}
int main() {
ios_base::sync_with_stdio(false); cin.tie(NULL);
sieve();
cin >> n >> q;
for (int i = 1; i <= n; i++) {
int val; cin >> val;
// Tách thừa số nguyên tố cho a[i]
while (val > 1) {
int p = spf[val];
if (primes_of[i].empty() || primes_of[i].back() != p)
primes_of[i].push_back(p);
while (val % p == 0) val /= p;
}
}
for (int i = 1; i < n; i++) {
int u, v; cin >> u >> v;
adj[u].push_back(v); adj[v].push_back(u);
}
dfs(1);
B = timer / sqrt(q) + 1;
vector<Query> Q(q);
for (int i = 0; i < q; i++) {
int u, v; cin >> u >> v;
if (st[u] > st[v]) swap(u, v);
int lca = getLCA(u, v);
if (lca == u) {
Q[i] = {st[u], st[v], -1, i}; // Không cần LCA
} else {
Q[i] = {en[u], st[v], lca, i}; // Nhớ gắn thêm LCA
}
}
sort(Q.begin(), Q.end());
vector<int> ans(q);
int L = 1, R = 0;
for (Query qry : Q) {
while (L > qry.l) toggle(E[--L]);
while (R < qry.r) toggle(E[++R]);
while (L < qry.l) toggle(E[L++]);
while (R > qry.r) toggle(E[R--]);
int temp_lca = qry.lca;
if (temp_lca != -1) toggle(temp_lca); // Thêm LCA vào đường đi
ans[qry.id] = cur_ans;
if (temp_lca != -1) toggle(temp_lca); // Xong thì bỏ LCA ra
}
for (int i = 0; i < q; i++) cout << ans[i] << "\n";
return 0;
}

Độ phức tạp

  • Thời gian:
    Sàng nguyên tố mất O(M \log \log M).
    Phân tích các ai mất O(N \log A).
    Duyệt DFS + LCA cấu trúc bảng thưa: O(N \log N).
    Thuật toán Mo: Con trỏ L, R trượt tối đa O(N \sqrt{Q}) bước. Mỗi bước thao tác toggle mất O(k) (với k là số ước nguyên tố tối đa của 105, k \le 7).
    → Tổng cộng O(k × N \sqrt{Q} + N \log N). Với N, Q = 50000, N \sqrt{Q} ≈ 1.1 × 107, tổng phép toán khoảng 8 \times 107. Hoàn toàn vừa vặn vượt qua giới hạn thời gian 1-2 giây!
  • Không gian bộ nhớ:
    Mảng LCA 17 × 50000, mảng Euler 100000, Mảng đếm nguyên tố cnt 100000.
    → Tổng bộ nhớ: O(N \log N + \max(a_i)). Chiếm dưới 10MB, cực kỳ tối ưu.

Cạm bẫy khi cài đặt (Edge Cases)

✕ Lỗi dễ bỏ sót

Quên xử lý đỉnh LCA! Khi u không phải tổ tiên của v, đoạn Euler từ EN[u] đến ST[v] sẽ chưa hề chứa LCA của u và v. Nếu bỏ qua bước này sẽ đếm thiếu các nguyên tố nằm tại đỉnh LCA.

✓ Cách xử lý đúng

Trước khi ghi nhận đáp án cur_ans, kiểm tra nếu Query có LCA riêng (khác u). Ta gọi toggle(LCA) một lần để tạm bật nó. Ghi đáp án xong, phải toggle(LCA) lại lần nữa để tắt nó đi, không làm hỏng trạng thái cho các Query sau.

✕ Lỗi Time Limit

Sử dụng std::set hoặc std::map để đếm số lượng nguyên tố phân biệt trong hàm toggle. Cấu trúc cây đỏ đen có hằng số rất lớn, làm O(\log N) dội lên nhiều lần dẫn tới TLE.

✓ Cách xử lý đúng

Các nguyên tố đều ≤ 10^{5}. Hãy dùng mảng tĩnh cnt[100005] làm bản đồ băm (Hash map) hoàn hảo với truy xuất O(1). Cập nhật biến nguyên cur_ans khi có phần tử thay đổi giữa mức 0 và mức 1.
