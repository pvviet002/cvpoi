cin/cout · if/else
vòng lặp for · mảng 1 chiều
Bấm nút để xem một dãy 12 ô gấp lại thành bảng 3 hàng × 4 cột.
Dãy thẳng: 12 ô nằm liên tiếp, đánh số từ 0 đến 11 — đúng như mảng 1 chiều em đã học.
Phần 1

Vì sao cần mảng 2 chiều?

Mảng 1 chiều lưu được một dãy. Nhưng nhiều thứ trong đời thực là một bảng.

1 Bài toán: bảng điểm của lớp

Lớp có 3 học sinh, mỗi bạn có 4 điểm kiểm tra. Nếu dùng mảng 1 chiều, em phải khai báo 3 mảng riêng:

int an[4], binh[4], cuong[4];  // 30 bạn thì sao?!

Không ổn. Ta cần một cấu trúc lưu được cả hàng (mỗi bạn một hàng) và cột (mỗi bài kiểm tra một cột).

2 Lời giải: một bảng duy nhất

int diem[3][4];  // 3 hàng, 4 cột = 12 ô

// Điểm bài 3 của bạn thứ 2:
diem[1][2] = 9;

Đọc là: “ô ở hàng 1, cột 2”. Chỉ số vẫn đếm từ 0, y như mảng 1 chiều.

  • Bàn cờ caro → char ban[10][10];
  • Ảnh đen trắng → int anh[100][100];
  • Thời khoá biểu → int tkb[6][10];
Phần 2

Khai báo và truy cập

3 Khai báo

// Cú pháp:  kiểu  tên[số_hàng][số_cột];
int  a[3][4];       // 3 hàng, 4 cột
char c[8][8];       // bàn cờ vua

// Khai báo kèm giá trị luôn:
int b[2][3] = {
    {1, 2, 3},   // hàng 0
    {4, 5, 6}    // hàng 1
};

Mỗi cặp ngoặc nhọn bên trong là một hàng. Nhìn vào code là thấy luôn hình dạng cái bảng.

Mẹo thi: nếu chưa biết trước kích thước, cứ khai báo dư rồi dùng một phần:

const int MAX = 100;
int a[MAX][MAX];   // dùng m hàng, n cột thôi
int m, n;
cin >> m >> n;

4 Truy cập một ô

Với int b[2][3] ở bên trái:

ViếtNghĩaGiá trị
b[0][0]hàng 0, cột 01
b[0][2]hàng 0, cột 23
b[1][0]hàng 1, cột 04
b[1][2]hàng 1, cột 26

Chỉ số đầu luôn là hàng, chỉ số sau luôn là cột. Nhớ câu này: “hàng trước, cột sau”.

Mảng a[m][n] có hàng hợp lệ từ 0 đến m-1, cột từ 0 đến n-1.

Phần 4

Nhập, xuất và xử lý

Quy tắc vàng: muốn đụng tới mọi ô của bảng thì cần hai vòng for lồng nhau.

5 Nhập bảng từ bàn phím

int a[100][100], m, n;
cin >> m >> n;

for (int i = 0; i < m; i++)
    for (int j = 0; j < n; j++)
        cin >> a[i][j];

6 In bảng ra màn hình

for (int i = 0; i < m; i++) {
    for (int j = 0; j < n; j++)
        cout << a[i][j] << " ";
    cout << endl;   // xuống dòng khi hết 1 hàng
}

Dòng cout << endl; đặt ngoài vòng for của j — nếu đặt trong, mỗi số sẽ nằm một dòng.

7 Ba bài toán kinh điển

Tính tổng tất cả phần tử

int tong = 0;
for (int i = 0; i < m; i++)
    for (int j = 0; j < n; j++)
        tong += a[i][j];

Đếm số chẵn (dùng if đã học)

int dem = 0;
for (int i = 0; i < m; i++)
    for (int j = 0; j < n; j++)
        if (a[i][j] % 2 == 0)
            dem++;

Tìm giá trị lớn nhất

int maxV = a[0][0];
for (int i = 0; i < m; i++)
    for (int j = 0; j < n; j++)
        if (a[i][j] > maxV)
            maxV = a[i][j];

Thấy không? Bộ khung hai vòng for luôn giống hệt nhau — chỉ đổi phần việc ở giữa.

Phần 5 · Kỹ thuật

Cờ hiệu và lính canh

Hai mẹo nhỏ nhưng có mặt trong gần như mọi bài mảng. Cả hai đều chỉ là… một biến được đặt đúng chỗ.

8 Cờ hiệu (flag) — biến để ghi nhớ

Vòng lặp không tự nhớ được chuyện gì đã xảy ra bên trong nó. Vậy ta nuôi một biến bool làm “người ghi sổ”:

bool timThay = false;   // hạ cờ: chưa xảy ra
// ... trong vòng lặp, nếu gặp chuyện:
timThay = true;            // kéo cờ lên
// ... sau vòng lặp, nhìn cờ mà kết luận:
if (timThay) ... else ...
  • Cờ chỉ có 2 trạng thái: false (chưa) → true (rồi).
  • Đã kéo lên thì không bao giờ hạ xuống trong cùng một lần duyệt.
  • Đặt timThay = false trước vòng lặp, kiểm tra sau vòng lặp.

Vì sao cần cờ ở mảng 2 chiều? Vì lệnh break chỉ thoát một vòng lặp — vòng trong. Muốn thoát luôn vòng ngoài, phải hỏi lại cây cờ.

9 Lính canh (sentinel) — giá trị chắc chắn thua

Khi đi tìm số lớn nhất, biến max phải bắt đầu từ đâu đó. Nếu khởi tạo bừa bằng 0 thì:

int maxV = 0;   // BẪY!
// Mảng toàn số âm: -5 -9 -2
// Kết quả in ra 0 — một số KHÔNG hề có trong mảng.

Lính canh là một giá trị đứng gác ở vạch xuất phát, nhỏ hơn mọi giá trị có thể có, để chắc chắn nó sẽ bị thay ngay ở phần tử đầu tiên:

const int VOCUC = -1000000000;  // lính canh
int max1 = VOCUC;

// Hoặc an toàn nhất: lấy luôn phần tử đầu tiên
int max1 = a[0][0];

Phần thưởng: sau vòng lặp, nếu biến vẫn còn là lính canh thì nghĩa là chẳng có gì thay nó cả → trường hợp “không tồn tại”. Một biến làm hai việc.

BÀI 6

Cho mảng 2 chiều và một số nguyên X nhập từ bàn phím. Hãy tìm xem X có xuất hiện trong mảng không. Nếu có, in ra toạ độ (i, j) của lần xuất hiện đầu tiên.

Kỹ năng: tìm kiếm tuyến tính trên lưới 2D · cờ hiệu flag · break để dừng sớm.

Cách 1 — cờ hiệu + hai lệnh break

bool timThay = false;
int hang, cot;

for (int i = 0; i < m; i++) {
    for (int j = 0; j < n; j++) {
        if (a[i][j] == X) {
            timThay = true;
            hang = i;  cot = j;
            break;          // thoát vòng j
        }
    }
    if (timThay) break;   // thoát vòng i
}

if (timThay)
    cout << hang << " " << cot;
else
    cout << "Khong tim thay";

Chú ý dòng if (timThay) break; — thiếu nó, chương trình vẫn cho đáp án đúng nhưng phải chạy hết cả bảng một cách vô ích.

Cách 2 — nhét cờ vào điều kiện lặp

bool timThay = false;
int hang, cot;

for (int i = 0; i < m && !timThay; i++)
    for (int j = 0; j < n && !timThay; j++)
        if (a[i][j] == X) {
            timThay = true;
            hang = i;  cot = j;
        }

Đọc !timThay là “chưa tìm thấy”. Vòng lặp chỉ tiếp tục khi cờ còn hạ. Ngắn hơn, không cần break nào cả.

Cách 3 — dùng luôn lính canh, bỏ hẳn cờ

int hang = -1, cot = -1;   // lính canh
for (int i = 0; i < m && hang == -1; i++)
    for (int j = 0; j < n && hang == -1; j++)
        if (a[i][j] == X) { hang = i; cot = j; }

if (hang == -1) cout << "Khong tim thay";
else cout << hang << " " << cot;

Toạ độ -1 là bất khả thi, nên nó vừa là kho chứa kết quả, vừa là cờ hiệu.

BÀI 9

Cho mảng 2 chiều, hãy tìm phần tử có giá trị phân biệt lớn thứ hai trong mảng.

Kỹ năng: hai biến lính canh max1, max2 · hoặc dồn mảng 2D thành 1D rồi sắp xếp.

“Phân biệt” nghĩa là gì?

Mảng 9 9 7 5 có lớn nhất là 9. Lớn thứ hai phân biệt là 7, chứ không phải số 9 thứ hai. Ta chỉ quan tâm tới các giá trị khác nhau, không quan tâm chúng xuất hiện mấy lần.

Ý tưởng: đừng làm hai việc cùng lúc

Tìm hạng nhất và hạng nhì trong một vòng lặp rất dễ rối. Vậy ta tách hẳn ra hai lần duyệt, mỗi lần chỉ lo đúng một chuyện — và cả hai lần đều dùng đúng một khuôn if tìm max mà em đã quen:

Lần duyệtNhiệm vụĐiều kiện trong if
Lần 1 Tìm giá trị lớn nhất, gọi là max1. a[i][j] > max1
Lần 2 Tìm giá trị lớn nhất trong số những ô nhỏ hơn max1. Đó chính là max2. a[i][j] < max1 && a[i][j] > max2

Điều kiện a[i][j] < max1 ở lần 2 chính là chỗ chữ “phân biệt” nằm: nó gạch bỏ mọi ô bằng đúng max1, kể cả những ô 9 lặp lại. Nếu viết nhầm thành <=, máy sẽ tìm lại chính max1 và in ra 9 — sai đề.

Đổi lại ta chạy chậm gấp đôi. Với bảng 100×100 thì 10 000 ô duyệt 2 lần vẫn chỉ là 20 000 phép so sánh — máy tính làm trong tích tắc. Dễ hiểu và chắc đúng quan trọng hơn nhanh.

Cách 1 — duyệt hai lần (khuyên dùng)

const int VOCUC = -1000000000;

// ===== LẦN 1: tìm giá trị lớn nhất =====
int max1 = VOCUC;
for (int i = 0; i < m; i++)
    for (int j = 0; j < n; j++)
        if (a[i][j] > max1)
            max1 = a[i][j];

// ===== LẦN 2: lớn nhất trong những số < max1 =====
int max2 = VOCUC;
for (int i = 0; i < m; i++)
    for (int j = 0; j < n; j++)
        if (a[i][j] < max1 && a[i][j] > max2)
            max2 = a[i][j];

if (max2 == VOCUC)
    cout << "Khong co gia tri lon thu hai";
else
    cout << max2;

Hai khối code giống nhau như hai giọt nước, chỉ khác đúng một điều kiện trong if. Không có phép “dồn max1 xuống max2”, không sợ viết nhầm thứ tự.

Vai trò hai lính canh: max1 = VOCUC để phần tử đầu tiên chắc chắn vượt qua; max2 = VOCUC vừa làm vạch xuất phát, vừa làm cờ hiệu — sau lần 2 mà nó vẫn còn nguyên thì cả bảng chỉ có một giá trị duy nhất (ví dụ 5 5 5 5), tức là không tồn tại số lớn thứ hai.

Cách 2 — dồn 2D về 1D rồi sort

Bảng vốn đã là một dãy thẳng trong bộ nhớ (nhớ lại phần đầu bài), nên ta cứ chép nó ra một mảng 1 chiều rồi nhờ hàm sort có sẵn của C++ sắp xếp hộ:

#include <algorithm>   // nơi có hàm sort
using namespace std;

int b[10000], k = 0;

// đổ cả bảng vào mảng 1 chiều b
for (int i = 0; i < m; i++)
    for (int j = 0; j < n; j++)
        b[k++] = a[i][j];   // k chạy 0,1,2,...

// sắp xếp GIẢM dần: lớn nhất nằm ở b[0]
sort(b, b + k, greater<int>());

// đi từ đầu, tìm số ĐẦU TIÊN khác b[0]
int kq = VOCUC;
for (int p = 1; p < k; p++)
    if (b[p] != b[0]) { kq = b[p]; break; }

if (kq == VOCUC) cout << "Khong co";
else cout << kq;
  • sort(b, b + k) sắp xếp tăng dần đoạn từ b[0] đến b[k-1]. Viết b + k chứ không phải b + k - 1.
  • Thêm greater<int>() để đổi thành giảm dần. Khi đó b[0] chính là max1.
  • Vòng for cuối lại dùng break — gặp số khác b[0] đầu tiên là dừng ngay. Cờ hiệu ở đây là kq với lính canh VOCUC.

So sánh: viết nhanh, ít lỗi hơn tự code sắp xếp. Nhưng tốn thêm một mảng phụ 10 000 phần tử và sort vẫn chậm hơn hai vòng duyệt ở Cách 1. Trong bài thi có giới hạn thời gian, Cách 1 vẫn là lựa chọn tốt hơn.

Phần 6

Bốn lỗi ai cũng mắc lần đầu

✕ Vượt quá chỉ số

int a[3][3];
a[3][0] = 5;   // SAI!

Mảng a[3][3] chỉ có hàng 0, 1, 2. Không hề có hàng 3. Chương trình có thể vẫn chạy nhưng cho kết quả bậy hoặc bị crash.

✓ Điều kiện lặp đúng

for (int i = 0; i < m; i++)   // i < m, không phải i <= m

Luôn là < chứ không phải <=, vì chỉ số cuối cùng là m-1.

✕ Nhầm hàng với cột

a[j][i]   // đảo ngược mất rồi!

a[1][2] và a[2][1] là hai ô khác nhau. Nhớ: biến của vòng for ngoài đi vào ngoặc đầu.

✕ Viết a[i, j]

a[i, j]     // SAI trong C++
a[i][j]     // ĐÚNG

C++ bắt buộc mỗi chỉ số một cặp ngoặc vuông riêng.
