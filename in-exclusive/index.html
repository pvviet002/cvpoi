Bao hàm — loại trừ

Phan Văn Việt — Trường THPT Chuyên Vĩnh Phúc

Đếm số phần tử thỏa ít nhất một trong nhiều điều kiện: cộng vào rồi trừ ra theo số điều kiện chẵn lẻ, để không phần tử nào bị đếm hai lần.

Ý tưởng chủ đạo

Để tính kích thước của tập hợp hợp (union), ta cộng tổng các tập hợp đơn lẻ, trừ đi phần giao của các cặp, cộng lại phần giao của các bộ ba, trừ đi phần giao của bộ bốn... Cứ thế đan dấu.

Dùng khi nào?

Bài toán đếm số lượng thỏa mãn ít nhất một trong K điều kiện, với K nhỏ (thường K ≤ 20), nhưng không gian đếm N lại rất lớn (ví dụ N = 1012).

Điểm mạnh

Giảm độ phức tạp từ việc duyệt từng phần tử O(N) xuống thành duyệt qua tất cả các cấu hình tập hợp O(2K).

Điểm yếu

Số lượng điều kiện (K) phải đủ nhỏ vì chi phí tăng theo hàm mũ 2K. Nếu K lớn (như K = 105), PIE sẽ thất bại do tràn thời gian.

2.Bài toán và động lực

Giả sử bạn cần đếm xem trong khoảng từ 1 đến N (ví dụ N = 109), có bao nhiêu số chia hết cho 2, chia hết cho 3, hoặc chia hết cho 5.

Cách ngây thơ: Dùng vòng lặp for (i = 1 → N), kiểm tra từng số. Rõ ràng sẽ bị Time Limit Exceeded (TLE) vì 109 phép toán mất ~1 giây, chưa kể kiểm tra nhiều điều kiện.

Khó khăn then chốt: Ta biết số lượng số chia hết cho 2 là N / 2, chia hết cho 3 là N / 3. Nhưng nếu ta chỉ cộng lại: (N / 2) + (N / 3) + (N / 5) thì sẽ đếm thừa. Những số như 6 (chia hết cho cả 2 và 3) đã bị cộng 2 lần. Những số như 30 bị cộng tới 3 lần!

Ý tưởng cứu vãn: Dùng sơ đồ Venn. Cộng tất cả cái đơn lẻ. Bị trùng phần giao của 2 tập? Trừ nó đi. Lại lỡ trừ quá tay phần giao của 3 tập? Cộng nó lại. Đó chính là Nguyên lý Bao hàm - Loại trừ (Principle of Inclusion-Exclusion - PIE).

Công thức cốt lõi

Cho các tập hợp A1, A2, ..., AK. Kích thước của tập hợp hợp (union) được tính bởi:

|A1 ∪ A2 ∪ ... ∪ AK| = Tổng các tập 1 - Tổng giao 2 + Tổng giao 3 - ... + (-1)K-1 Tổng giao K

Dấu của biểu thức: Phụ thuộc vào số lượng tập hợp đang tham gia vào phần giao.
- Nếu chọn lẻ tập (1, 3, 5...): dấu Cộng (+)
- Nếu chọn chẵn tập (2, 4, 6...): dấu Trừ (-)

Vì sao đúng? Bất kỳ phần tử nào thuộc đúng X tập hợp (X ≥ 1) sẽ đóng góp vào tổng bằng: C(X,1) - C(X,2) + C(X,3) - ... = 1 (theo khai triển nhị thức Newton của (1-1)X). Tức là nó luôn được đếm chính xác 1 lần!

3.Thuật toán từng bước

Để duyệt qua mọi cách chọn (giao) của K tập hợp, công cụ mạnh nhất là Bitmask (mặt nạ bit). Một số nguyên từ 1 đến 2K - 1 sẽ biểu diễn một cấu hình chọn.

Thuật toán cài đặt PIE

  1. Chuẩn bị: Cho mảng primes gồm K phần tử (các điều kiện). Khởi tạo kết quả ans = 0.
  2. Duyệt mặt nạ: Cho biến mask chạy từ 1 đến (1 << K) - 1 (tương đương 2K - 1). Mỗi bit 1 trong mask ứng với việc chọn phần tử đó vào tập giao.
  3. Trích xuất thông tin từ mask:
    • Đếm số bit 1 của mask (có thể dùng __builtin_popcount(mask)).
    • Xác định dấu: sign = (số bit lẻ) ? 1 : -1.
  4. Tính phần giao:
    • Khởi tạo biến lcm = 1.
    • Duyệt i từ 0 đến K-1. Nếu bit thứ i của mask đang bật, cập nhật lcm = LCM(lcm, primes[i]).
    • Lưu ý: Dừng sớm nếu lcm > N để tránh tràn số.
  5. Cập nhật kết quả: Số lượng phần tử thuộc tập giao này là N / lcm. Cộng vào đáp án: ans += sign * (N / lcm).

4.Cài đặt C++

long long solve_PIE(long long N, vector<long long>& primes) { int K = primes.size(); long long ans = 0; // Duyệt qua 2^K - 1 trạng thái (bỏ qua trạng thái 0 vì rỗng) for (int mask = 1; mask < (1 << K); ++mask) { long long current_lcm = 1; int bits_count = 0; for (int i = 0; i < K; ++i) { // Kiểm tra bit thứ i có bật không if ((mask >> i) & 1) { bits_count++; // Tính BCNN. Chú ý: current_lcm * primes[i] / GCD long long g = std::gcd(current_lcm, primes[i]); // Tránh tràn số (overflow) if (current_lcm / g > N / primes[i]) { current_lcm = N + 1; // Gán một số lớn hơn N break; } current_lcm = (current_lcm / g) * primes[i]; } } long long count_elements = N / current_lcm; // Quy tắc đan dấu: lẻ cộng, chẵn trừ if (bits_count % 2 == 1) { ans += count_elements; } else { ans -= count_elements; } } return ans; }

7.Mở rộng và lỗi thường gặp

Lỗi: Tràn số khi tính LCM

Triệu chứng: Bị WA (Wrong Answer) với N lớn. Tích của các số nguyên tố/điều kiện trong một mask có thể vượt quá giới hạn 64-bit (long long).

Cách tránh: Thay vì viết lcm *= p, hãy dùng gcd. Trái tim của việc chống tràn nằm ở đoạn code: if (lcm / g > N / p) break; (Dừng vì dù sao N/lcm cũng bằng 0).

Lỗi: Chạy mask từ 0

Triệu chứng: Lặp vô tận hoặc sai kết quả (Chia cho 0 hoặc đếm sai rỗng).

Cách tránh: Mask 0 biểu diễn "không chọn tập nào". Về mặt PIE cổ điển, ta chỉ giao các tập hợp, nên số lượng tập ≥ 1. Luôn cho mask chạy từ 1. Tuy nhiên, nếu tìm "số lượng không thỏa mãn đ/k nào", ta lấy Tổng mẫu trừ đi PIE.

Ứng dụng: Bài toán Hoán vị sai (Derangements)

Có N người mang N chiếc mũ tới tiệc. Lúc về, mất điện, mỗi người vơ đại 1 mũ. Hỏi có bao nhiêu cách để không ai lấy đúng mũ của mình?

Sử dụng PIE: Đặt Ai là tập các cách mà người i lấy đúng mũ của mình. Ta cần tìm: N! - |A1 ∪ A2 ∪ ... ∪ AN|.
Kết quả cuối cùng rút gọn cực đẹp: N! × (1/0! - 1/1! + 1/2! - 1/3! + ... + (-1)N/N!).
