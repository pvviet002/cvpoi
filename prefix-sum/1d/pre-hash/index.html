Ý tưởng chủ đạo

Phân tích tổng đoạn [L, R] thành hiệu hai tiền tố: SR - SL-1 = K. Biến đổi thành SL-1 = SR - K. Ta dùng std::map để lưu trữ lịch sử các Si đã gặp, từ đó biết được SL-1 có tồn tại hay không chỉ trong O(log N).

Dùng khi nào?

Khi bài toán yêu cầu Đếm số lượng, Tìm độ dài Max/Min của đoạn con liên tiếp có tổng bằng K, tổng chia hết cho K, hoặc các điều kiện cân bằng (số lượng 0 bằng số lượng 1).

Điểm mạnh & Yếu

Mạnh: Khử vòng lặp lồng nhau, xử lý tốt mảng có số âm và tổng cực lớn.
Yếu: Khai báo std::map mang theo hằng số thời gian (overhead) khá lớn so với mảng tĩnh.

2.Bài toán và động lực

Yêu cầu: Cho mảng A có N phần tử (có thể chứa số âm). Đếm số lượng đoạn con liên tiếp có tổng đúng bằng K.

Cách tiếp cận ngây thơ (Naive) sẽ cố định điểm đầu L, duyệt điểm cuối R và cộng dồn dần để kiểm tra. Cách này đòi hỏi xét mọi cặp (L, R), dẫn đến O(N²).

Quan sát then chốt:
Nếu ta có mảng cộng dồn S, tổng đoạn từ L đến R là S[R] - S[L-1].
Thay vì hỏi: "Từ L này, có R nào tạo ra tổng K không?" (Duyệt tiến tới)
Ta đổi góc nhìn tại R: "Tại R này, liệu trước đó đã có S[L-1] nào thỏa mãn S[L-1] = S[R] - K không?" (Nhìn về quá khứ).

Vì mảng cộng dồn S có thể chứa số âm, tổng lên tới 1014, ta không thể đếm phân phối bằng mảng tĩnh count[10000]. Cấu trúc std::map<long long, int> trong C++ sinh ra để giải quyết hoàn hảo vấn đề này: lưu trữ mọi giá trị (khóa) dù lớn hay âm, và đếm tần suất của chúng trong O(log N).

Thuật toán từng bước: Đếm đoạn con tổng K

  1. Khởi tạo dữ liệu cơ sở:
    • Khai báo biến ans = 0, biến tính tổng tạm sum = 0.
    • Khai báo Hash Map mp lưu tần suất.
    • Bắt buộc: Gán mp[0] = 1. Giả lập một trạng thái tiền tố bằng 0 ở vị trí trước phần tử đầu tiên. (Để nếu đoạn [1, R] có tổng K, ta có S[R] - 0 = K).
  2. Duyệt mảng (i từ 1 đến N):
    • Cộng dồn giá trị hiện tại: sum += a[i]. (Bản chất sum chính là Si).
  3. Tính mục tiêu và tra cứu:
    • Tính giá trị tiền tố cần tìm: target = sum - K.
    • Nếu target đã tồn tại trong map mp, ta cộng dồn số lần xuất hiện của nó vào ans. Thực hiện: ans += mp[target].
  4. Ghi nhận lịch sử:
    • Tăng tần suất của giá trị sum hiện tại vào map để các phần tử phía sau có thể tra cứu: mp[sum]++.

3.Cài đặt C++ — đếm đoạn con tổng K

long long countSubarraysSumK(vector<int>& a, long long k) { long long ans = 0, sum = 0; map<long long, int> mp; mp[0] = 1; // Khởi tạo mốc cơ sở: tiền tố rỗng có tổng = 0 for (int i = 0; i < a.size(); i++) { sum += a[i]; // Tính S[R] long long target = sum - k; // S[L-1] cần tìm if (mp.count(target)) { ans += mp[target]; } mp[sum]++; // Cập nhật S[R] vào map } return ans; }

4.Đoạn con dài nhất có tổng K

Thay vì đếm số lượng, ta cần tối đa hóa độ dài đoạn (R - L + 1). Vẫn chung ý tưởng tra cứu target = S[R] - K.

Quan sát: Để đoạn [L, R] dài nhất, chỉ số L-1 (tức điểm bắt đầu của tiền tố bị cắt đi) phải nhỏ nhất có thể (càng xa về bên trái càng tốt). Do đó, trong Hash Map, thay vì lưu tần suất, ta lưu vị trí xuất hiện đầu tiên của từng giá trị S[i].

int maxSubarrayLenSumK(vector<int>& a, long long k) { long long sum = 0; int maxLen = 0; map<long long, int> mp; mp[0] = -1; // S[0]=0 đứng ở vị trí TRƯỚC mảng (index -1) for (int i = 0; i < a.size(); i++) { sum += a[i]; long long target = sum - k; if (mp.count(target)) { maxLen = max(maxLen, i - mp[target]); // Độ dài = R - (L-1) } // CHỈ lưu vị trí nếu sum CHƯA từng xuất hiện // Để giữ vị trí nhỏ nhất (xa nhất về trái) if (!mp.count(sum)) { mp[sum] = i; } } return maxLen; }

Mô phỏng 2: Tìm đoạn con dài nhất tổng K = 2

Vẫn dùng mảng A = [2, -1, 3, -2, 4]. Mục tiêu đổi thành tìm chiều dài Max. Map bây giờ lưu (Sum → Vị trí đầu tiên).

5.Các lỗi thường gặp

Quên khởi tạo mp[0] = 1

Triệu chứng: Bỏ sót các đoạn con bắt đầu từ vị trí đầu tiên (L=1). Chạy thử tay mẫu nhỏ sẽ thấy đếm thiếu.

Cách tránh: Luôn nạp base-case vào Map ngay khi khai báo. Hãy coi như có phần tử vô hình ở index -1 mang tổng 0.

Toán tử Module % âm trong C++

Triệu chứng: Khi giải bài "Đoạn con chia hết cho K", -2 % 5 trong C++ ra -2 thay vì 3. Tra map sai hoàn toàn.

Cách tránh: Ép số dư luôn dương bằng công thức: rem = ((sum % K) + K) % K;.

Bị TLE do Map Overhead

Triệu chứng: std::map mất O(log N) mỗi thao tác. Với N = 5×10⁵, đôi khi bị Time Limit Exceeded.

Cách tránh: Dùng std::unordered_map (O(1) trung bình). Tuy nhiên trên Codeforces phải viết thêm Custom Hash, nếu không sẽ bị hack chèn test đụng độ hash làm giảm xuống O(N).
