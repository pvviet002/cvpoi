📝 Tóm tắt & Hướng giải

Luật: Camera (2) phát tia (3) sang phải. Dừng khi gặp mép hoặc vật cản (1). Đếm ô trống (0).

Cách làm: Quét mảng. Gặp camera -> chạy vòng lặp mới sang phải, đổi ô trống thành tia đến khi gặp cản thì break.

Cú pháp & Logic

for (int i = 0; i < N; i++) {
for (int j = 0; j < M; j++) {
if (a[i][j] == 2) { // Gặp Camera
for (int k = j + 1; k < M; k++) {
if (a[i][k] == 1) break; // Chạm cản -> Ngừng
if (a[i][k] == 0) a[i][k] = 3; // Đánh dấu tia
}
}
}
}

Phòng Thí Nghiệm

Bước: 0/0
Bấm "Tiến" để bắt đầu.
i (hàng): -
j (cột_cam): -
k (cột_tia): -
Safe Cells: 0
for (i = 0; i < 4; i++) {
for (j = 0; j < 6; j++) {
if (a[i][j] == CAMERA) {
for (k = j + 1; k < 6; k++) {
if (a[i][k] == OBS) break;
if (a[i][k] == EMPTY) a[i][k] = RAY;
}
}
}
}

⚠️ Cạm Bẫy

✕ Quên lệnh break

Nếu không có if (a[i][k] == 1) break;, code sẽ bỏ qua vật cản và tiếp tục quét tia hồng ngoại xuyên thấu một cách vô lý.

🧠 Kiểm Tra Nhanh

Vì sao xét tia bắt đầu từ k = j + 1?

Chính xác! Cột j chứa camera. Tia hồng ngoại phát từ camera sẽ bắt đầu quét từ ô liền kề bên phải, tức là j + 1.
