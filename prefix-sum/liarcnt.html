1.Tại sao cần tư duy lại?

Bài toán yêu cầu tìm một tọa độ x sao cho số người nói dối là ít nhất.

Tọa độ x có thể là số thực bất kỳ! Nếu ta thử chạy vòng lặp kiểm tra từng giá trị x = 1, 1.1, 1.2, … thì sẽ không bao giờ xong (Vô hạn).

💡 Điểm mấu chốt: Sự thật/Dối trá chỉ thay đổi khi x vượt qua các điểm mốc (các giá trị p được cung cấp trong input). Giữa các mốc, số lượng người nói dối là không đổi!

Do đó, ta chỉ cần thử các ứng viên x chính là các tọa độ p đã cho.

Định nghĩa "Kẻ Nói Dối"

  • L p (Khai x ≤ p):
    Sẽ là nói dối NẾU thực tế Cuội ở x > p.
    👉 Viết ngược lại: Khi ta chọn tọa độ x, những người khai `L p` mà có p < x là kẻ nói dối!
  • G p (Khai x ≥ p):
    Sẽ là nói dối NẾU thực tế Cuội ở x < p.
    👉 Viết ngược lại: Khi ta chọn tọa độ x, những người khai `G p` mà có p > x là kẻ nói dối!

2.Thuật toán cốt lõi và tối ưu

Ta sẽ thu thập tất cả các tọa độ xuất hiện trong Input vào một mảng `points`. Do sự thật chỉ thay đổi ở các "cột mốc" nên ta chỉ cần đặt Cuội vào từng vị trí x trong `points`.

Cách 1: Duyệt mảng (Subtask 1)

Kiểm tra từng x, rồi lặp qua toàn bộ mảng L và G để đếm. Độ phức tạp là O(N2), chỉ phù hợp N ≤ 1000.

int min_liars = INF;
for (int x : points) {
    // Đếm trâu (vòng lặp)
    int lie_L = count_if(L, p < x);
                    min_liars = min(min_liars, lie_L + lie_G);
}

Cách 2: Mảng cộng dồn (Prefix Sum)

Lọc các tọa độ duy nhất và sắp xếp thành mảng U. Dùng mảng cộng dồn đếm số lượng khai L ≤ x và mảng hậu tố đếm G ≥ x. Tránh việc duyệt lại mảng nhiều lần, cho phép tính số người nói dối tại mọi mốc trong O(1).

// 1. Tiền xử lý mảng cộng dồn và hậu tố
for(int i = 0; i < m; ++i) {
    pref_L[i] = (i > 0 ? pref_L[i-1] : 0) + count_L[U[i]];
}
for(int i = m - 1; i >= 0; --i) {
    suff_G[i] = (i < m-1 ? suff_G[i+1] : 0) + count_G[U[i]];
}

// 2. Tính số lượng nói dối tại từng mốc
int min_liars = INF;
for (int i = 0; i < m; ++i) {
    int lie_L = (i > 0 ? pref_L[i-1] : 0); // Số người nói L có p < x
    int lie_G = (i < m-1 ? suff_G[i+1] : 0); // Số người nói G có p > x
    
    min_liars = min(min_liars, lie_L + lie_G);
}

4.Cạm bẫy thường gặp

✕ Sai lầm

Nhầm lẫn dấu bằng khi kiểm tra điều kiện nói dối.

// L p nghĩa là Cuội <= p.
// KHI NÀO là nói dối?
if (p <= x) { lie_L++; } // SAI!

Nếu x = p, thì Cuội ở tại p, mệnh đề x ≤ p vẫn là SỰ THẬT, không phải nói dối.

✓ Cách Đúng

Phủ định logic cẩn thận. Mệnh đề x ≤ p sai khi x > p.

// L p sai khi thực tế x > p
// (hay p < x)
if (p < x) { lie_L++; } // ĐÚNG!

Chỉ đếm khi mốc p nhỏ hơn hẳn x (đối với `L`), hoặc lớn hơn hẳn x (đối với `G`).

5.Kiểm tra nhanh

Câu 1: Xử lý mệnh đề mâu thuẫn

Giả sử có 1 người khai L 10 và 1 người khai G 10. Nếu ta đoán Cuội đang trốn ở đúng tọa độ x = 10, có bao nhiêu người trong số 2 người này nói dối?

Câu 2: Dấu vết của Cuội

Bờm nhận được 3 thông tin:
1. Bạn A nói: G 3 (Cuội ở x ≥ 3)
2. Bạn B nói: L 5 (Cuội ở x ≤ 5)
3. Bạn C nói: G 8 (Cuội ở x ≥ 8)

Nếu Bờm quyết định lục soát tại bụi cây ở vị trí x = 4, ai là kẻ nói dối trong tình huống này?

Câu 3: Vận dụng mảng cộng dồn

Trong thuật toán Mảng cộng dồn (Prefix Sum), công thức tính số người khai L nói dối (điều kiện p < x) tại tọa độ x = U[i] là gì?
