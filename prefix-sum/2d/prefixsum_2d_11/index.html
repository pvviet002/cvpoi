Độ khó: Khó (O(N2 · M))

Đếm Hình Chữ Nhật Chia Hết Cho K

Bản chất: Cố định hai biên hàng để nén ma trận thành dãy 1D. Sau đó, biến việc đếm mảng con chia hết thành việc đếm các tiền tố có cùng số dư bằng Mảng tần số.

Tóm tắt đề bài

Cho ma trận lưới kích thước N × M và số nguyên K. Cần đếm số lượng hình chữ nhật con có tổng các phần tử bên trong chia hết cho K.

  • Ràng buộc: N, M ≤ 300; K ≤ 106; Ai,j ≤ 109.
  • Ví dụ: Lưới 2×2, K = 3.
1 2
2 1

Có 5 hình chữ nhật thỏa mãn tổng chia hết cho 3 (ví dụ: [1 2], [2 1], [1 2 dọc], [2 1 dọc], [cả 4 ô = 6]).

Phân tích ngây thơ

Tương tự như các bài toán đếm hình chữ nhật khác, nếu ta dùng 4 vòng lặp để duyệt tọa độ (r1, c1) và (r2, c2), sau đó kiểm tra Tổng % K == 0, ta sẽ mất O(N2 · M2). Với N, M = 300, số lượng phép tính lên tới 3004 = 8.1 × 109. Chắc chắn sẽ bị quá thời gian (TLE).

Để tối ưu, ta phải giảm số vòng lặp xuống còn 3 bằng cách nén 2D thành 1D.

1. Kỹ thuật Ép 2D thành 1D

Khác với việc duyệt 4 biến, ta chủ động cố định biên trên r1 và biên dưới r2 (bằng 2 vòng lặp). Khi đã cố định được chiều dọc, ta tính tổng của từng cột nằm kẹp giữa r1 và r2.

Ngay lúc này, bức tranh 2D đã xẹp xuống thành một Dãy số 1D. Bài toán quy về: Cho một mảng 1D, đếm số đoạn con có tổng chia hết cho K.

Hành động "tính tổng cột" này nên được thực hiện như thế nào để đạt O(1) cho mỗi cột?
Giải thích: Chính xác! Bằng công cụ mảng cộng dồn 2D đã học, ta lấy phần hình chữ nhật dài từ hàng 1 đến r2 trừ đi phần thừa phía trên (từ hàng 1 đến r1-1). Việc này tạo ra dãy 1D ngay lập tức trong O(1).

2. Tính chất Chia hết và Mảng đếm dư

Bây giờ, ta cần đếm số mảng con 1D có tổng chia hết cho K. Gọi mảng cộng dồn của dãy 1D này là P. Tổng đoạn từ cột c1 đến c2 là P[c2] - P[c1-1].

Điều kiện để đoạn này chia hết cho K là:

(P[c2] - P[c1-1]) % K == 0

Về mặt toán học, điều này tương đương với:

P[c2] % K == P[c1-1] % K

Nghĩa là: Ta chỉ việc tính tiền tố P duyệt dần theo từng cột. Nếu cột hiện tại có số dư mod, ta chỉ cần cộng vào đáp án số lần số dư này đã xuất hiện trước đó (bằng một mảng đếm tần số cnt).

Tại sao trước khi duyệt cột đầu tiên, ta BẮT BUỘC phải mồi sẵn giá trị cnt[0] = 1?
Giải thích: Chính xác! Gốc tọa độ tưởng tượng (trước khi cộng số nào) có tổng bằng 0, nên nó mang số dư 0. Ta giả vờ như nó đã xuất hiện 1 lần. Nếu cột c hiện tại cũng có dư 0, nó có thể ghép với gốc tọa độ đó để tạo thành một hình chữ nhật chia hết cho K.

3. Tối ưu cực độ thời gian Reset

Mảng đếm tần số cnt có kích thước bằng K. Theo đề bài, K có thể lên tới 106.

Mỗi khi xét một cặp hàng (r1, r2) mới, ta phải "làm sạch" mảng cnt về 0. Tuy nhiên, nếu dùng hàm memset hay for(int i=0; i<K; i++), ta sẽ mất thao tác xóa 106 phần tử cho MỖI cặp hàng. Có tới (300×300)/2 = 45,000 cặp hàng, tổng thao tác làm sạch là 45,000 × 106 = 4.5 × 1010. Chắc chắn sẽ Quá thời gian (TLE)!

Ý Tưởng: Xóa dấu vết thông minh

Ta nhận thấy mỗi cặp hàng có độ dài cột M=300. Dù có cập nhật thế nào, mảng cnt cũng chỉ bị thay đổi tối đa 300 vị trí. Thay vì lau dọn toàn bộ tòa nhà, ta chỉ cần nhớ lại những chỗ đã làm bẩn và lau đúng chỗ đó!

Sử dụng một std::vector<int> used;. Mỗi khi một số dư mod xuất hiện lần đầu (cnt[mod] == 0), ta ném nó vào giỏ used. Xong việc, ta lôi giỏ ra: for(int x : used) cnt[x] = 0;. Phép toán dọn dẹp giảm từ O(K) xuống chỉ còn O(M).

Công thức nấu ăn (Thuật toán từng bước)

  1. Chuẩn bị tiền xử lý:
    • Nhập lưới và xây dựng Mảng cộng dồn 2D S[N][M] kiểu long long.
    • Khai báo mảng cnt[K] toàn cục (mặc định = 0). KHÔNG xóa nó bên trong vòng lặp.
  2. Cố định hai biên hàng ngang:
    • Duyệt r1 từ 1 đến N, và r2 từ r1 đến N.
    • Bên trong, tạo giỏ lưu dấu vết: vector<int> used;
  3. Mồi số 0 và đếm 1D:
    • Gán cnt[0] = 1; và used.push_back(0);.
    • Duyệt cột c từ 1 đến M:
      • Lấy tổng khối lượng vắt qua cột này: val = S[r2][c] - S[r1-1][c].
      • Tính phần dư: mod = val % K.
      • Cộng vào đáp số: ans += cnt[mod].
      • Ghi lại dấu vết nếu là lần đầu thấy: if(cnt[mod] == 0) used.push_back(mod).
      • Tăng tần số: cnt[mod]++.
  4. Dọn dẹp vết bẩn cực nhanh:
    • Lặp qua các phần tử trong giỏ: for(int x : used) cnt[x] = 0;
    • Kết thúc cặp hàng, mảng cnt lại sạch trơn để đón cặp hàng mới.

Cài đặt (C++)

#include <iostream>
#include <vector>
using namespace std;
// S dùng long long vì tổng các ô có thể vượt mức 2 tỷ
long long S[305][305];
int cnt[1000005]; // Mảng đếm tần số dư, K tối đa 10^6
int main() {
ios_base::sync_with_stdio(false);
cin.tie(NULL);
int N, M;
long long K;
if (!(cin >> N >> M >> K)) return 0;
// 1. Tiền xử lý mảng cộng dồn 2D
for (int i = 1; i <= N; i++) {
for (int j = 1; j <= M; j++) {
long long val;
cin >> val;
S[i][j] = S[i-1][j] + S[i][j-1] - S[i-1][j-1] + val;
}
}
long long ans = 0;
// 2. Cố định 2 biên hàng r1 và r2
for (int r1 = 1; r1 <= N; r1++) {
for (int r2 = r1; r2 <= N; r2++) {
// 3. Khởi tạo mảng đếm tần số cho dải cột 1D
vector<int> used;
cnt[0] = 1; // Rất quan trọng: mồi sẵn giá trị 0
used.push_back(0); // Ghi nhớ đã dùng số 0
for (int c = 1; c <= M; c++) {
long long P_c = S[r2][c] - S[r1-1][c];
int mod = P_c % K;
// Cộng dồn tần số phần dư vào đáp án
ans += cnt[mod];
// Ghi vết để reset sau này (nếu chưa có trong cnt)
if (cnt[mod] == 0) {
used.push_back(mod);
}
cnt[mod]++;
}
// 4. Xóa chính xác các phần tử đã dùng để tránh TLE
for (int x : used) {
cnt[x] = 0;
}
}
}
cout << ans << "\n";
return 0;
}

Độ phức tạp

  • Thời gian: Có N2/2 cặp hàng. Với mỗi cặp hàng, ta duyệt qua M cột mất O(M). Các phép reset dọn mảng chạy đúng số lần các cột xuất hiện, cũng chỉ tốn O(M). Tổng thời gian cực kỳ ổn định là O(N2 · M). Cụ thể 3003 = 2.7 × 107 thao tác, chạy rất nhanh.
  • Không gian bộ nhớ: O(N · M) cho mảng 2D và O(K) cho mảng cnt. Khoảng vài MB, cực kỳ an toàn.

Cạm bẫy khi cài đặt

✕ Reset thủ công toàn mảng

Dùng memset(cnt, 0, sizeof(cnt)) ở đầu mỗi cặp hàng.

Vì mảng có 1 triệu phần tử, dọn sạch nó 90,000 lần sẽ vắt kiệt CPU, tạo ra TLE ngay lập tức.

✓ Dùng danh sách lưu vết bẩn

Dùng vector<int> used để lưu lại những phần dư nào đã được tăng lên.

Hành động này giống như "cầm giẻ lau thẳng vào vết bẩn" thay vì cọ rửa lại cả căn nhà.

✕ Không mồi cnt[0]

Vào vòng lặp đếm luôn không cần thiết lập mồi.

Bạn sẽ bị sót toàn bộ các hình chữ nhật kéo dài từ cạnh ngoài cùng bên trái. Đếm thiếu số lượng đáng kể.

✓ Luôn mồi gốc tọa độ

Gán cnt[0] = 1 và used.push_back(0) trước khi lặp.

Tương đương việc chốt chặn cột số 0 có số dư 0.
