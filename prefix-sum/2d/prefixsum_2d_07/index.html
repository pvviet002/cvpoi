Độ khó: Trung bình (O(26 · N · M + 26 · Q))

Sự Đa Dạng Của Bức Tranh

Bản chất: Chuyển bài toán "đếm số màu" trong một tập hợp nhỏ (26 ký tự) thành 26 bài toán "kiểm tra sự tồn tại" riêng biệt, xử lý bằng Mảng cộng dồn 2D.

Tóm tắt đề bài

Cho một lưới N × M với mỗi ô là một ký tự từ 'a' đến 'z'. Với mỗi truy vấn dạng (r1, c1, r2, c2), yêu cầu đếm xem có bao nhiêu ký tự (màu sắc) khác nhau xuất hiện trong vùng chữ nhật đó.

  • Ràng buộc: N, M ≤ 300; Q ≤ 105.
  • Ví dụ: Vùng có các ký tự a, b, a, c, c thì trả về 3 (vì có 3 loại: a, b, c).

Phân tích ngây thơ

Cách đơn giản nhất là với mỗi truy vấn, dùng 2 vòng lặp lồng nhau duyệt qua mọi ô trong hình chữ nhật đó. Dùng một mảng boolean đánh dấu hoặc std::set để đếm các ký tự khác nhau.

Tại sao không đủ tốt? Một vùng truy vấn có thể lớn bằng cả bức tranh (300 × 300 = 90,000 ô). Nếu có 105 truy vấn, tổng số phép duyệt lên tới 9 × 109, sẽ gây ra lỗi Quá thời gian (TLE). Chúng ta cần công cụ O(1) để trả lời truy vấn. Mảng cộng dồn 2D sinh ra để làm điều này, nhưng nó chỉ tính tổng chứ không đếm tập hợp.

1. Ý tưởng: Tách lớp màu sắc

Một điểm mấu chốt: Bảng màu rất nhỏ (chỉ có 26 chữ cái). Thay vì hỏi câu hỏi khó "Trong vùng này có bao nhiêu màu?", ta có thể hỏi 26 câu hỏi dễ: "Màu 'a' có ở trong vùng này không?", "Màu 'b' có không?"... rồi đếm số câu trả lời CÓ.

Với một ký tự cụ thể k (ví dụ 'a'), ta có thể tưởng tượng việc "chiếu tia X" vào bức tranh chỉ để nhìn thấy chữ 'a'. Ta tạo một mảng 2D mới, trong đó các ô có chữ 'a' biến thành số 1, còn chữ khác biến thành số 0.

Một khi đã có mảng Boolean 1/0 này, ta chỉ việc tính Mảng Cộng Dồn 2D cho riêng lớp đó (như bài Tổng hình chữ nhật con). Bằng cách này, ta sở hữu tận 26 mảng cộng dồn 2D độc lập. Ta ký hiệu Sk[i][j] là mảng cộng dồn của ký tự thứ k.

Khảo sát mảng cộng dồn của màu:
Bức tranh
Lớp 1/0
Mảng S của màu này
Tại sao cách tách lớp này chỉ khả thi với các bài toán có tập hợp phần tử nhỏ (ví dụ 26 chữ cái)?
Giải thích: Mảng 3D để lưu các lớp là S[Số_lượng_màu][N][M]. Với N, M = 300, nếu chỉ có 26 màu, mảng cần 26 × 300 × 300 (khoảng 2.3 triệu phần tử) -> rất an toàn. Nhưng nếu có số lượng màu tuỳ ý (ví dụ số nguyên lên đến 105 loại), bộ nhớ cần sẽ là 100000 × 300 × 300, gây vỡ bộ nhớ (Memory Limit Exceeded) ngay lập tức!

2. Giải quyết truy vấn

Giả sử ta nhận được truy vấn: Đếm số màu trong vùng từ (r1, c1) đến (r2, c2).

Ta sẽ tạo một biến đếm ans = 0. Sau đó, dùng một vòng lặp duyệt qua 26 màu (từ k = 0 đến 25):

  1. Sử dụng mảng Sk, tính số lần xuất hiện của màu k trong vùng truy vấn bằng công thức:

    cnt = Sk[r2][c2] - Sk[r1-1][c2] - Sk[r2][c1-1] + Sk[r1-1][c1-1]

  2. Nếu cnt > 0 (nghĩa là màu k có mặt), ta tăng ans++.

Bằng cách này, mỗi truy vấn chỉ mất 26 phép tính, tức là O(26) ≈ O(1).

Tại sao ta lại đếm bằng lệnh if (cnt > 0) ans++ thay vì lấy ans += cnt?
Giải thích: Chính xác! Biến cnt cho biết số lượng ký tự của một màu trong vùng (ví dụ có 3 chữ 'a' thì cnt = 3). Nhưng nếu ta muốn biết "Sự đa dạng của bức tranh" (số LOẠI màu), ta chỉ quan tâm xem màu đó "có tồn tại không" (tức là cnt > 0). Sự tồn tại đó sẽ đóng góp đúng 1 đơn vị vào tổng số loài màu.

Công thức nấu ăn (Thuật toán từng bước)

  1. Khai báo và Đọc dữ liệu:
    • Khai báo mảng 3 chiều S[26][N+1][M+1].
    • Đọc vòng lặp qua i (1 đến N) và j (1 đến M). Với mỗi ô, đọc ký tự c.
  2. Tiền xử lý (Tính cộng dồn khi đang đọc):
    • Dùng vòng lặp màu k chạy từ 0 đến 25.
    • Xác định xem ô hiện tại có phải màu k không: Nếu c - 'a' == k, gán val = 1, ngược lại val = 0.
    • Tính mảng cộng dồn cho lớp k: S[k][i][j] = S[k][i-1][j] + S[k][i][j-1] - S[k][i-1][j-1] + val.
  3. Xử lý truy vấn:
    • Với mỗi truy vấn (r1, c1, r2, c2), đặt ans = 0.
    • Lặp k từ 0 đến 25:
      • Tính số lần màu k xuất hiện: cnt = S[k][r2][c2] - S[k][r1-1][c2] - S[k][r2][c1-1] + S[k][r1-1][c1-1].
      • Nếu cnt > 0, tăng ans++.
    • In ra ans.

Cài đặt (C++)

#include <iostream>
#include <vector>
using namespace std;
// S[màu][hàng][cột]
int S[26][305][305];
int main() {
// Tối ưu I/O cho số lượng truy vấn lớn
ios_base::sync_with_stdio(false);
cin.tie(NULL);
int N, M, Q;
if (!(cin >> N >> M >> Q)) return 0;
// Bước 1 & 2: Xây dựng 26 lớp mảng cộng dồn
for (int i = 1; i <= N; i++) {
for (int j = 1; j <= M; j++) {
char c;
cin >> c;
int color_idx = c - 'a';
for (int k = 0; k < 26; k++) {
int val = (k == color_idx) ? 1 : 0;
S[k][i][j] = S[k][i-1][j] + S[k][i][j-1] - S[k][i-1][j-1] + val;
}
}
}
// Bước 3: Trả lời truy vấn
while (Q--) {
int r1, c1, r2, c2;
cin >> r1 >> c1 >> r2 >> c2;
int ans = 0;
// Duyệt kiểm tra sự tồn tại của 26 màu độc lập
for (int k = 0; k < 26; k++) {
int cnt = S[k][r2][c2] - S[k][r1-1][c2] - S[k][r2][c1-1] + S[k][r1-1][c1-1];
if (cnt > 0) {
ans++;
}
}
cout << ans << "\n";
}
return 0;
}

Độ phức tạp

  • Thời gian:
    • Xây dựng mảng: Mất 26 vòng lặp cho mỗi ô → O(26 · N · M).
    • Truy vấn: Mất 26 vòng lặp kiểm tra cho mỗi truy vấn → O(26 · Q).
    • Tổng thời gian: O(26 · N · M + 26 · Q). Với N, M = 300 (khoảng 90,000 ô) và Q = 105, tổng số phép tính khoảng 2.3 triệu + 2.6 triệu ≈ 5 triệu thao tác. Chạy cực nhanh (chỉ khoảng 0.01 giây trong C++).
  • Không gian bộ nhớ: Cần mảng 3D kích thước 26 × 305 × 305 số nguyên. Kích thước khoảng ~9.6 MB, hoàn toàn thỏa mãn giới hạn bộ nhớ thông thường (thường là 256 MB).

Cạm bẫy khi cài đặt

✕ Thứ tự mảng không tối ưu

Khai báo int S[305][305][26].

Mặc dù vẫn đúng kết quả, nhưng việc để lớp màu ở chiều cuối cùng làm cho bộ nhớ không liên tục khi duyệt mảng theo cùng một màu, gây chậm do hiệu ứng "Cache miss".

✓ Lớp màu ở chiều đầu tiên

Khai báo int S[26][305][305].

Tương đương với việc ta có 26 mảng 2D hoàn toàn tách biệt. Khi truy cập S[k], toàn bộ mảng của lớp k nằm liền kề nhau trong bộ nhớ, truy xuất rất mượt mà.

✕ In sai số lượng

Cộng dồn đáp án: ans += cnt;

Như đã phân tích ở Quiz 2, bạn sẽ đếm tổng toàn bộ số ký tự trong vùng thay vì đếm số loại màu sắc.

✓ Đếm sự tồn tại

Luôn bọc trong điều kiện: if (cnt > 0) ans++;.

Chỉ cần màu sắc có xuất hiện (ít nhất 1 pixel) là đã tính là một loại màu trong độ phong phú của bức tranh.
