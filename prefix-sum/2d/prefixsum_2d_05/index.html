Độ khó: Khó (O(N2 · M))

Đếm Hình Chữ Nhật Có Tổng Bằng S

Bản chất: Bằng cách cố định hai biên hàng ngang, ta biến bài toán đếm hình chữ nhật 2D phức tạp thành một bài toán tìm mảng con 1D có tổng S, giải quyết cực nhanh bằng Hash Map.

Tóm tắt đề bài

Cho ma trận N × M và một số nguyên S. Cần đếm xem có bao nhiêu hình chữ nhật con có tổng các phần tử bên trong đúng bằng S.

  • Ràng buộc: N, M ≤ 100; Giá trị phần tử ≤ 1000.
  • Ví dụ: Lưới 3×3, S = 5.
1 2 2
3 -1 1
0 5 -5

Có 4 hình chữ nhật có tổng bằng 5 (ví dụ: hàng 1 cột 1 đến 3 có 1+2+2=5; hoặc đoạn chứa mỗi số 5 ở hàng 3).

Phân tích ngây thơ

Dựa vào bài học Mảng cộng dồn 2D, ta có thể tính nhanh tổng một hình chữ nhật chỉ trong O(1). Do đó, cách nghĩ đầu tiên là duyệt qua mọi tọa độ góc trái trên (r1, c1) và góc phải dưới (r2, c2), sau đó kiểm tra xem tổng có bằng S không.

long long ans = 0;
for (int r1 = 1; r1 <= N; r1++) {
for (int c1 = 1; c1 <= M; c1++) {
for (int r2 = r1; r2 <= N; r2++) {
for (int c2 = c1; c2 <= M; c2++) {
long long sum = S[r2][c2] - S[r1-1][c2] - S[r2][c1-1] + S[r1-1][c1-1];
if (sum == Target) ans++;
}
}
}
}

Tại sao cần cải tiến? Lời giải này dùng 4 vòng lặp lồng nhau, độ phức tạp là O(N2 · M2). Với N, M = 100, số vòng lặp tối đa là 108, vẫn có thể chạy qua (tốn khoảng 0.1 giây trong C++). Nhưng nếu đề bài mở rộng N, M lên 300, 1004 = 8.1 × 109 sẽ dẫn đến TLE ngay lập tức. Chúng ta cần một phương pháp thông minh hơn.

1. Ý tưởng cốt lõi: Nén 2D thành 1D

Hãy nhìn kỹ lại công thức tính tổng 2D ban đầu:

Sum = S[r2][c2] - S[r1-1][c2] - S[r2][c1-1] + S[r1-1][c1-1]

Thay vì duyệt 4 biến cùng lúc, ta nhóm các phần có chung chỉ số cột lại với nhau:

Sum = (S[r2][c2] - S[r1-1][c2]) - (S[r2][c1-1] - S[r1-1][c1-1])

Giả sử ta chủ động CỐ ĐỊNH biên trên r1 và biên dưới r2. Khi đó, cụm S[r2][c] - S[r1-1][c] chỉ còn phụ thuộc vào biến cột c. Đặt giá trị đó là P[c]. Công thức sẽ rút gọn thành:

Sum = P[c2] - P[c1-1]

Thật bất ngờ! P[c2] - P[c1-1] chính xác là công thức tính tổng mảng con trên 1 chiều (1D). Điều này có nghĩa: một khi đã cố định độ cao r1 và r2, ta đã "nén" ma trận 2D thành một dãy số 1D.

Thay đổi thanh trượt để quan sát mảng P được sinh ra.
Giá trị P[c] = S[r2][c] - S[r1-1][c] mang ý nghĩa hình học là gì?
Giải thích: Theo nguyên lý bao hàm loại trừ của mảng 2D, S[r2][c] tính từ góc (1,1) đến (r2, c). Khi trừ đi phần thừa phía trên S[r1-1][c], ta sẽ giữ lại đúng dải hình chữ nhật từ hàng r1 tới r2, chiều rộng ngang từ cột 1 đến cột c.

2. Giải bài toán 1D bằng Hash Map

Với mảng P vừa sinh ra, ta cần đếm số cặp cột (c1, c2) sao cho P[c2] - P[c1-1] = S.

Nếu làm ngây thơ duyệt 2 vòng lặp, ta sẽ mất O(M2). Nhưng ta có thể làm trong O(M) bằng biến đổi đại số:

P[c1-1] = P[c2] - S

Khi duyệt dần biến c2 từ 1 đến M, ta coi c2 là đích đến. Tại mỗi bước, ta chỉ cần hỏi: "Trong các cột c1-1 trước đây, đã có bao nhiêu cột đạt giá trị bằng đúng (P[c2] - S)?". Ta dùng một Hash Map (Bảng băm) để ghi nhớ tần số xuất hiện của các giá trị P quá khứ.

Tại sao BẮT BUỘC phải khởi tạo map[0] = 1 trước khi bắt đầu vòng lặp duyệt cột?
Giải thích: Nếu một hình chữ nhật hợp lệ kéo dài từ tận biên trái (cột 1) đến c2, tổng của nó chính là P[c2] (hoặc P[c2] - P[0]). Do đó lượng cần tìm req = P[c2] - S sẽ bằng 0. Nếu không mồi sẵn giá trị 0 vào map, đoạn dính biên trái này sẽ bị bỏ sót hoàn toàn!

Công thức nấu ăn (Thuật toán từng bước)

  1. Chuẩn bị Mảng cộng dồn 2D:
    • Khai báo mảng 2 chiều S[N+1][M+1].
    • Tính như bình thường: S[i][j] = S[i-1][j] + S[i][j-1] - S[i-1][j-1] + A[i][j].
  2. Duyệt ép 2D thành 1D (Cố định hàng):
    • Dùng 2 vòng lặp r1 từ 1 đến N, và r2 từ r1 đến N.
    • Bên trong vòng r2, ta chuẩn bị giải quyết một bài toán con 1D độc lập.
  3. Đếm số mảng con 1D bằng Hash Map:
    • Khai báo bảng băm map (kiểu dữ liệu unordered_map<long long, int>).
    • Khởi tạo hạt giống: map[0] = 1.
    • Duyệt vòng lặp cột c từ 1 đến M:
      • Lấy tổng phần cột tương ứng: P_c = S[r2][c] - S[r1-1][c].
      • Lượng quá khứ cần tìm: req = P_c - Target.
      • Nếu req có trong map, cộng tần số của nó vào tổng biến đếm ans.
      • Lưu vết: Tăng tần số của P_c trong map lên 1.

Cài đặt (C++)

#include <iostream>
#include <vector>
#include <unordered_map>
using namespace std;
const int MAXN = 105;
long long A[MAXN][MAXN];
long long S[MAXN][MAXN];
int main() {
ios_base::sync_with_stdio(false);
cin.tie(NULL);
int N, M;
long long Target;
if (!(cin >> N >> M >> Target)) return 0;
// 1. Đọc dữ liệu và xây dựng mảng cộng dồn 2D
for (int i = 1; i <= N; i++) {
for (int j = 1; j <= M; j++) {
cin >> A[i][j];
S[i][j] = S[i-1][j] + S[i][j-1] - S[i-1][j-1] + A[i][j];
}
}
long long ans = 0;
// 2. Cố định 2 biên hàng r1 và r2
for (int r1 = 1; r1 <= N; r1++) {
for (int r2 = r1; r2 <= N; r2++) {
// 3. Kỹ thuật đếm Hash Map trên 1D
unordered_map<long long, int> prefix_count;
prefix_count[0] = 1; // Cực kỳ quan trọng!
for (int c = 1; c <= M; c++) {
// Dải cột từ r1 đến r2, tính qua mảng 2D
long long P_c = S[r2][c] - S[r1-1][c];
long long req = P_c - Target;
if (prefix_count.find(req) != prefix_count.end()) {
ans += prefix_count[req];
}
// Lưu P_c vào map để duyệt các c2 tiếp theo
prefix_count[P_c]++;
}
}
}
cout << ans << "\n";
return 0;
}

Độ phức tạp

  • Thời gian: Có 2 vòng lặp lồng nhau duyệt hàng (tốn N2 / 2 bước). Bằng mỗi cặp hàng, ta duyệt qua M cột mất O(M) nhờ unordered_map tra cứu trong O(1). Tổng thời gian O(N2 · M). Với lưới lớn nhất 100 × 100, số thao tác cực nhỏ (khoảng 500,000), dễ dàng vượt qua thời gian giới hạn 1 giây.
  • Không gian bộ nhớ: O(N · M) để lưu ma trận S. Bên trong vòng lặp dùng thêm một map chứa tối đa M phần tử. Nhìn chung rất tiết kiệm.

Cạm bẫy khi cài đặt

✕ Quên map[0] = 1

Khởi tạo unordered_map trống rỗng.

Hệ quả là bạn sẽ bị đếm hụt TẤT CẢ các hình chữ nhật có cột bắt đầu từ sát lề bên trái (tức c1 = 1). Chương trình không bắt được đoạn nào khớp mốc 0 cả.

✓ Gieo hạt giống map

Luôn viết prefix_count[0] = 1; ngay trước vòng lặp duyệt cột.

Điều này cho biết: trước khi ghép cột nào, ta coi như có sẵn tổng 0 tại gốc tọa độ ảo (cột 0).

✕ Dùng chung một Hash Map

Khai báo prefix_count tuốt bên ngoài các vòng lặp r1, r2 để đỡ tốn thời gian tạo mới.

Cực sai! Bảng băm sẽ bị nhiễm dữ liệu của những lần cố định hàng trước đó. Kết quả ans sẽ cộng dồn loạn xạ sinh ra con số khổng lồ vô lý.

✓ Khởi tạo Map ĐỘC LẬP

Khai báo biến map NẰM BÊN TRONG phạm vi của vòng lặp r2.

Nhờ thế mỗi khi chuyển qua một cặp hàng r1-r2 mới, bài toán được reset hoàn toàn sạch sẽ để đếm lại 1D từ đầu.
