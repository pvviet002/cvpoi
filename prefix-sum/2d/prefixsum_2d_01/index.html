Độ khó: Trung bình (O(N·M + Q))

Tổng hình chữ nhật con

Bản chất: Trả lời nhanh truy vấn tính tổng một vùng con 2D bất kỳ bằng cách mượn ý tưởng "bù trừ" diện tích hình học (Nguyên lý bao hàm - loại trừ).

Tóm tắt đề bài

Cho một lưới kích thước N × M chứa các số nguyên. Có Q truy vấn, mỗi truy vấn yêu cầu tính tổng các số trong một hình chữ nhật con giới hạn bởi góc trên trái (r1, c1) và góc dưới phải (r2, c2).

  • Ràng buộc: N, M ≤ 1000; Q ≤ 105; Giá trị phần tử ≤ 109.
  • Ví dụ: Lưới 3×4. Yêu cầu tính tổng vùng từ (2,2) đến (3,4).
    Các phần tử trong vùng đó là: 6, 7, 8 (hàng 2) và 10, 11, 12 (hàng 3). Tổng = 54.

Phân tích ngây thơ

Cách dễ nhất để tính tổng là duyệt qua mọi phần tử trong vùng được yêu cầu:

long long sum = 0;
for (int i = r1; i <= r2; i++) {
for (int j = c1; j <= c2; j++) {
sum += A[i][j];
}
}

Tại sao không đủ tốt? Với mỗi truy vấn, ta có thể phải lặp qua toàn bộ N × M ô. Tổng thời gian cho Q truy vấn là O(N · M · Q). Với N, M = 1000 và Q = 105, số phép tính lên tới 1011 (cần hơn 100 giây), chắc chắn bị quá thời gian (TLE).

Chúng ta cần một cấu trúc dữ liệu cho phép trả lời mỗi truy vấn chỉ trong O(1).

1. Ý tưởng: Mở rộng mảng cộng dồn 1D

Ở mảng 1D, để tính tổng đoạn [L, R] nhanh, ta tạo mảng S[i] = tổng từ đầu đến i. Sau đó lấy S[R] - S[L-1].

Trong không gian 2D, ta mở rộng ý tưởng đó. Gọi S[i][j] là tổng của hình chữ nhật kéo dài từ góc trên trái (1, 1) đến ô hiện tại (i, j).

Nếu ta có mảng S này được tính sẵn, mọi truy vấn về sau sẽ chỉ là những phép cộng trừ các mảng lớn (O(1)). Câu hỏi đặt ra là: Làm sao xây dựng S, và làm sao dùng S để trả lời truy vấn?

2. Xây dựng mảng cộng dồn S[i][j]

Ta muốn tính S[i][j] dựa vào các ô S đã tính trước đó. Bằng trực giác hình học, tổng vùng từ (1,1) đến (i,j) bao gồm:

  1. Phần bên trên: S[i-1][j]
  2. Phần bên trái: S[i][j-1]
  3. Chú ý: Vùng giao nhau ở góc trên trái S[i-1][j-1] đã bị tính 2 lần (một lần trong phần trên, một lần trong phần trái). Ta phải trừ đi 1 lần.
  4. Cuối cùng, cộng thêm giá trị của chính ô hiện tại: A[i][j].

S[i][j] = S[i-1][j] + S[i][j-1] - S[i-1][j-1] + A[i][j]

Tại bước tính S[i][j], nếu ta KHÔNG trừ đi S[i-1][j-1] thì hệ quả là gì?
Giải thích: Vì S[i-1][j] (khối chữ nhật bên trên) và S[i][j-1] (khối chữ nhật bên trái) giao nhau tại vùng S[i-1][j-1]. Nên nếu ta chỉ lấy S[i-1][j] + S[i][j-1], vùng giao nhau đã được tính 2 lần (bị lặp). Trừ đi một lần là bắt buộc!

3. Lấy ra một hình chữ nhật con (Truy vấn)

Giả sử ta cần tính tổng vùng từ (r1, c1) đến (r2, c2). Ta đã có sẵn S. Quy trình cũng tương tự như lúc xây, nhưng là phép trừ:

  1. Lấy TOÀN BỘ hình chữ nhật từ (1,1) đến góc dưới phải: S[r2][c2].
  2. Cắt bỏ phần thừa bên trên: S[r1-1][c2].
  3. Cắt bỏ phần thừa bên trái: S[r2][c1-1].
  4. Lúc này, phần giao nhau ở góc trên trái (từ 1,1 đến r1-1, c1-1) vừa bị cắt bỏ đi tận 2 lần. Ta phải bù (cộng) lại 1 lần: S[r1-1][c1-1].

Tổng = S[r2][c2] - S[r1-1][c2] - S[r2][c1-1] + S[r1-1][c1-1]

Tại sao công thức cắt bỏ phần thừa bên trên lại dùng r1 - 1 (S[r1-1][c2]) mà KHÔNG phải là r1 (S[r1][c2])?
Giải thích: Truy vấn yêu cầu lấy từ hàng r1 đến r2. Hàng r1 là một phần của ĐÁP ÁN. Khối chữ nhật thừa nằm hoàn toàn phía trên nó chỉ kết thúc ở hàng r1 - 1. Do đó ta phải trừ đi S[r1-1][c2]. (Giống mảng 1D tính đoạn [L,R] phải lấy S[R] - S[L-1]).

Công thức nấu ăn (Thuật toán từng bước)

  1. Khai báo và Đọc dữ liệu:
    • Khai báo mảng 2D A[N+1][M+1] và S[N+1][M+1] (dùng mảng 1-based để tránh lỗi khi gọi chỉ số 0).
    • Lưu ý: Vì tổng có thể lên tới 1015, mảng S PHẢI dùng kiểu long long.
    • Đọc các giá trị vào A[i][j] với 1 ≤ i ≤ N, 1 ≤ j ≤ M.
  2. Khởi tạo biên mảng S bằng 0 (Tuỳ chọn nhưng an toàn):
    • Gán S[0][j] = 0 cho mọi j.
    • Gán S[i][0] = 0 cho mọi i.
    • (Trong C++, nếu khai báo mảng toàn cục, bước này được tự động làm).
  3. Xây dựng mảng cộng dồn S:
    • Dùng 2 vòng lặp lồng nhau: i chạy từ 1 đến N, j chạy từ 1 đến M.
    • Cập nhật: S[i][j] = S[i-1][j] + S[i][j-1] - S[i-1][j-1] + A[i][j].
  4. Xử lý các truy vấn:
    • Lặp qua Q truy vấn, đọc 4 số r1, c1, r2, c2.
    • In ra kết quả ngay lập tức: S[r2][c2] - S[r1-1][c2] - S[r2][c1-1] + S[r1-1][c1-1].

Cài đặt (C++)

#include <iostream>
#include <vector>
using namespace std;
const int MAXN = 1005;
long long A[MAXN][MAXN];
long long S[MAXN][MAXN]; // Mảng S phải dùng long long vì tổng rất lớn
int main() {
// Tối ưu I/O cho số lượng truy vấn lớn
ios_base::sync_with_stdio(false);
cin.tie(NULL);
int N, M, Q;
cin >> N >> M >> Q;
// 1. Đọc dữ liệu (1-based index)
for (int i = 1; i <= N; i++) {
for (int j = 1; j <= M; j++) {
cin >> A[i][j];
}
}
// 2. Tiền xử lý: Xây dựng mảng cộng dồn 2D
for (int i = 1; i <= N; i++) {
for (int j = 1; j <= M; j++) {
S[i][j] = S[i-1][j] + S[i][j-1] - S[i-1][j-1] + A[i][j];
}
}
// 3. Trả lời truy vấn
for (int q = 0; q < Q; q++) {
int r1, c1, r2, c2;
cin >> r1 >> c1 >> r2 >> c2;
long long ans = S[r2][c2] - S[r1-1][c2] - S[r2][c1-1] + S[r1-1][c1-1];
cout << ans << "\n";
}
return 0;
}

Độ phức tạp

  • Thời gian:
    • Bước tiền xử lý (xây mảng S): 2 vòng lặp lồng nhau mất O(N · M).
    • Bước truy vấn: mỗi truy vấn mất 4 phép tính cơ bản O(1), tổng cộng Q truy vấn mất O(Q).
    • Tổng thời gian: O(N · M + Q). Khoảng ~ 1.1 × 106 phép tính, chạy cực kỳ thoải mái trong giới hạn 1 giây của C++.
  • Không gian: Cần tạo thêm mảng S kích thước N × M. Không gian là O(N · M).

Cạm bẫy khi cài đặt

✕ Tràn số nguyên

Dùng int S[MAXN][MAXN].

Vì mảng N, M lên tới 1000, chứa tới 106 ô. Mỗi ô A[i][j] ≤ 109. Tổng toàn bộ bảng có thể là 1015, vượt xa mức 2 × 109 của kiểu int 32-bit (sẽ ra số âm vô lý).

✓ Dùng số nguyên 64-bit

Sử dụng long long S[MAXN][MAXN].

Kiểu long long trong C++ lưu được đến 9 × 1018, hoàn toàn đủ an toàn để chứa giá trị tổng 1015.

✕ Mảng 0-based

Lưu toạ độ từ 0. Khi truy vấn bị yêu cầu tính S[-1][c2] do r1 = 0.

Phải viết hàng loạt lệnh if (r1 > 0) rất mệt mỏi và dễ dính lỗi Out of Bounds (Truy cập mảng âm).

✓ Padding Mảng 1-based

Khai báo kích thước MAXN + 1. Đọc dữ liệu từ hàng 1, cột 1.

Khi đó viền 0 (hàng 0, cột 0) tự động là số 0. Khi r1 = 1, r1-1 bằng 0, mảng gọi S[0][c2] trả về 0 chính xác, không văng lỗi. Code cực sạch.
