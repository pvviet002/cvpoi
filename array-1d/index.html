Cạm bẫy số một: chỉ số bắt đầu từ 0
Bấm nút để đổi cách đánh nhãn, và bấm vào một ô để xem nó là phần tử thứ mấy.
Cách người thường đếm: phần tử thứ 1, 2, 3… Máy tính thì đếm từ 0, nên phần tử thứ k nằm ở a[k−1].
Phần 1

1.Vì sao cần mảng?

Khi phải lưu nhiều giá trị cùng loại, khai báo từng biến rời rạc là bất khả thi. Mảng cho ta một cái tên chung và một con số để chỉ vào từng ô.

1 Bài toán: điểm của 30 học sinh

Không dùng mảng, ta phải viết:

int d1, d2, d3, d4, d5, ... , d30;  // ác mộng

Muốn tính điểm trung bình phải cộng cả 30 biến bằng tay. Thêm một học sinh là sửa khắp chương trình.

2 Lời giải: một mảng duy nhất

int d[30];   // 30 ô, chỉ số 0..29

// cộng tất cả chỉ bằng một vòng lặp:
int tong = 0;
for (int i = 0; i < 30; i++)
    tong += d[i];

Một cái tên d, một chỉ số i, và mọi thao tác lặp lại gói gọn trong vòng for.

Phần 2

2.Khai báo và truy cập

3 Khai báo

// Cú pháp:  kiểu  tên[số_ô];
int  a[100];      // 100 số nguyên
double x[50];    // 50 số thực

// Khai báo kèm giá trị:
int b[5] = {4, 8, 15, 16, 23};

Mẹo thi: chưa biết trước số phần tử thì khai báo dư, rồi dùng biến n để đánh dấu phần thực sự dùng.

const int MAX = 100000;
int a[MAX], n;
cin >> n;   // chỉ dùng a[0..n-1]

4 Truy cập một ô

Với int b[5] = {4, 8, 15, 16, 23};:

Nhớ kỹ điều này

Mảng n phần tử có chỉ số hợp lệ từ 0 đến n−1. Ô cuối là a[n−1], không phải a[n]. Đây là lỗi phổ biến nhất với người mới.

3.Duyệt mảng bằng vòng lặp

Phần 4

4.Nhập, xuất và bài toán mẫu

5 Nhập và in mảng

int a[1000], n;
cin >> n;
for (int i = 0; i < n; i++)
    cin >> a[i];        // đọc n số vào mảng

for (int i = 0; i < n; i++)
    cout << a[i] << " ";  // in ra cách nhau dấu cách

Cùng một khuôn vòng for, chỉ đổi việc bên trong: khi thì cin, khi thì cout, khi thì tính toán.

6 Tìm max bằng lính canh

Đừng khởi tạo max = 0 — nếu mảng toàn số âm, kết quả sẽ ra 0 (một số không có trong mảng). Hãy lấy luôn phần tử đầu làm lính canh:

int maxV = a[0];   // lính canh an toàn
for (int i = 1; i < n; i++)
    if (a[i] > maxV)
        maxV = a[i];

Bắt đầu vòng lặp từ i = 1 vì a[0] đã dùng làm mốc so sánh. Đổi > thành < là được tìm min.

Phần 5 · Kỹ thuật

5.Hai kỹ thuật lõi

Tìm kiếm tuyến tính (với cờ hiệu) và hai con trỏ (để đảo mảng) — hai mẫu xuất hiện đi xuất hiện lại trong mọi bài mảng về sau.

TÌM KIẾM TUYẾN TÍNH

Cho mảng và số X. Tìm xem X có trong mảng không; nếu có, in ra vị trí đầu tiên gặp nó.

Kỹ năng: quét tuần tự · cờ hiệu flag hoặc lính canh -1 · break để dừng sớm.

Cách 1 — cờ hiệu + break

bool timThay = false;
for (int i = 0; i < n; i++)
    if (a[i] == X) {
        cout << i;
        timThay = true;
        break;      // gặp rồi, dừng ngay
    }
if (!timThay)
    cout << "Khong tim thay";

Không có break, chương trình vẫn đúng nhưng quét thừa hết mảng. Với mảng lớn và nhiều truy vấn, đó là lãng phí.

Cách 2 — lính canh thay cờ

int viTri = -1;   // lính canh: chưa thấy
for (int i = 0; i < n; i++)
    if (a[i] == X) { viTri = i; break; }

if (viTri == -1) cout << "Khong tim thay";
else cout << viTri;

Chỉ số -1 là bất khả thi, nên biến viTri vừa chứa kết quả vừa làm cờ hiệu. Một biến, hai việc.

ĐẢO NGƯỢC MẢNG

Cho mảng, hãy đảo thứ tự các phần tử: phần tử đầu thành cuối, cuối thành đầu.

Kỹ năng: hai con trỏ trai, phai từ hai đầu tiến vào giữa · kỹ thuật hoán đổi bằng biến tạm.

Ý tưởng hai con trỏ

Đặt một con trỏ ở đầu (trai = 0) và một ở cuối (phai = n−1). Đổi chỗ hai ô đó, rồi cho hai con trỏ tiến vào giữa. Dừng khi chúng gặp nhau.

int trai = 0, phai = n - 1;
while (trai < phai) {
    int tmp = a[trai];   // hoán đổi cần biến tạm
    a[trai] = a[phai];
    a[phai] = tmp;
    trai++;   phai--;      // tiến vào giữa
}

Muốn đổi chỗ hai cốc nước phải có cốc thứ ba — biến tmp chính là cái cốc đó. Nếu n lẻ, phần tử ở chính giữa đứng yên (khi trai == phai thì vòng lặp dừng).

Phần 6

6.Bốn lỗi thường gặp

✕ Truy cập vượt biên

int a[5];
a[5] = 9;   // SAI! ô hợp lệ là 0..4

Mảng 5 phần tử không có ô số 5. Truy cập a[5] đọc/ghi vào vùng nhớ lạ, cho kết quả bậy hoặc crash.

✓ Điều kiện lặp đúng

for (int i = 0; i < n; i++)  // i < n

Luôn là i < n, không phải i <= n. Chỉ số cuối cùng là n−1.

✕ Nghĩ chỉ số bắt đầu từ 1

a[1]  // đây là ô THỨ HAI, không phải ô đầu

Ô đầu tiên luôn là a[0]. Phần tử thứ k nằm ở a[k−1].

✕ Khởi tạo max = 0

int maxV = 0;  // sai khi mảng toàn số âm

Dùng maxV = a[0] làm lính canh để chắc chắn kết quả là một phần tử thật của mảng.
