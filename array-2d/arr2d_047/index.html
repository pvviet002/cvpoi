Tìm kiếm số hoàn hảo

Bí quyết của bài toán này không nằm ở vòng lặp ma trận, mà nằm ở sự hiểu biết về toán học đằng sau giới hạn của dữ liệu đầu vào.

Tóm tắt đề bài

Nhiệm vụ: Cho một ma trận kích thước N × M. Tìm số hoàn hảo lớn nhất xuất hiện trong ma trận. Nếu không có, in ra -1.

Số hoàn hảo: Là số nguyên dương bằng tổng các ước dương nhỏ hơn chính nó (ví dụ: 6 = 1 + 2 + 3).

Ràng buộc:
– Kích thước ma trận: 1 ≤ N, M ≤ 1000.
– Giá trị phần tử: 1 ≤ aij ≤ 109.

Ví dụ:

Input: 2 3 6 12 28 496 10 8 Output: 496

Phân tích ban đầu

Cách "ngây thơ" nhất mà chúng ta thường nghĩ tới: Duyệt qua từng ô của ma trận, và với mỗi giá trị X tại ô đó, ta dùng một vòng lặp từ 1 đến √X để tìm tất cả các ước số, cộng chúng lại và so sánh với X.

Vì sao cách này thất bại?

Ma trận có tối đa 1000 × 1000 = 106 phần tử. Trong trường hợp xấu nhất, mỗi phần tử đều có giá trị gần 109. Việc kiểm tra một số tốn khoảng √109 ≈ 31622 phép toán.

Tổng số phép toán sẽ là: 106 × 31622 ≈ 3 × 1010 phép toán. Con số này vượt xa ngưỡng cho phép thường thấy của các kỳ thi (khoảng 108 phép toán mỗi giây), và chắc chắn sẽ bị Quá thời gian (Time Limit Exceeded - TLE).

Vậy, ta cần một cách kiểm tra số hoàn hảo siêu tốc, tốn chi phí O(1) thay vì O(√X). Chìa khóa chính là sự thưa thớt của số hoàn hảo.

1. Sự thưa thớt của số hoàn hảo

Trong toán học, các số hoàn hảo vô cùng hiếm. Tính đến hiện tại, con người mới chỉ tìm ra khoảng hơn 50 số hoàn hảo, và khoảng cách giữa chúng tăng lên theo cấp số nhân.

Đề bài cho giới hạn phần tử aij ≤ 109. Hãy thử liệt kê xem có bao nhiêu số hoàn hảo nằm trong phạm vi này:

  • P1 = 6
  • P2 = 28
  • P3 = 496
  • P4 = 8128
  • P5 = 33550336
  • P6 = 8589869056 (Đã vượt quá 109)

Bất ngờ chưa? Chỉ có chính xác 5 số hoàn hảo có thể xuất hiện trong test của bài toán này! Thay vì phải mất công đi tìm ước và tính tổng cho từng số, ta chỉ cần so sánh giá trị aij xem nó có bằng một trong 5 con số "vàng" này hay không.

Mô phỏng: So sánh tốc độ kiểm tra X = 33550336
Cách Ngây Thơ (Lặp tìm ước) 0 phép toán
Cách Tối Ưu (So khớp mảng) 0 phép toán

❓ Dựa vào quan sát trên, nếu giới hạn của aij tăng lên đến 1018, phương pháp lưu sẵn danh sách số hoàn hảo (precomputation) còn khả thi hay không?

2. Duyệt ma trận và duy trì kết quả Max

Sau khi đã có "vũ khí" là danh sách 5 số hoàn hảo, việc còn lại rất đơn giản: Ta duyệt lần lượt qua các ô của ma trận 2 chiều.

Tại mỗi ô, ta kiểm tra xem nó có nằm trong danh sách không. Nếu có, ta cập nhật lại biến lưu trữ kết quả lớn nhất (giả sử đặt tên là ans).

Cạm bẫy khởi tạo: Đề bài yêu cầu in ra -1 nếu không tìm thấy số hoàn hảo nào. Do mọi số hoàn hảo đều là số dương (> 0), ta có thể khởi tạo giá trị ban đầu của ans = -1. Nếu duyệt hết ma trận mà ans vẫn là -1, nghĩa là không có số nào hợp lệ, ta in thẳng ans ra là chính xác.

Mô phỏng: Quá trình duyệt tìm Max (Mảng 2×3)
ans-1
i-
j-
long long ans = -1;
for (int i = 0; i < N; i++) {
for (int j = 0; j < M; j++) {
if (isPerfect(a[i][j])) {
ans = max(ans, a[i][j]);
}
}
}
return ans;
Nhấn Tiến hoặc Tự chạy để bắt đầu.

❓ Vì sao chúng ta dùng hàm kiểm tra từng số (isPerfect) thay vì lưu 5 số vào một mảng, rồi duyệt 5 số đó tìm xem nó có nằm trong ma trận không?

Thuật toán từng bước

  1. Chuẩn bị danh sách số hoàn hảo:
    • Tạo một tập hợp hoặc mảng chứa 5 số hoàn hảo hợp lệ: {6, 28, 496, 8128, 33550336}.
    • Viết một hàm isPerfect(x) để kiểm tra nhanh một số x có nằm trong danh sách này hay không.
  2. Khởi tạo biến theo dõi:
    • Khai báo biến ans = -1 để lưu số hoàn hảo lớn nhất tìm được. Lý do dùng -1 là để bao hàm luôn trường hợp không tìm thấy.
  3. Duyệt ma trận đầu vào:
    • Dùng 2 vòng lặp lồng nhau duyệt chỉ số dòng i từ 0 đến N-1, và chỉ số cột j từ 0 đến M-1.
    • Với mỗi phần tử a[i][j], gọi hàm isPerfect(a[i][j]).
    • Nếu hàm trả về true, thực hiện cập nhật: ans = max(ans, a[i][j]).
  4. Xuất kết quả:
    • Sau khi kết thúc vòng lặp, in giá trị của ans ra màn hình.

Cài đặt

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;
// Mảng chứa 5 số hoàn hảo duy nhất trong giới hạn 10^9
const long long PERFECT_NUMS[] = {6, 28, 496, 8128, 33550336};
// Hàm kiểm tra chi phí O(1)
bool isPerfect(long long x) {
for (int i = 0; i < 5; i++) {
if (x == PERFECT_NUMS[i]) return true;
}
return false;
}
int main() {
// Tối ưu I/O để đọc dữ liệu lớn nhanh hơn
ios_base::sync_with_stdio(false);
cin.tie(NULL);
int N, M;
if (!(cin >> N >> M)) return 0;
long long ans = -1; // Khởi tạo là -1 như bước phân tích
for (int i = 0; i < N; i++) {
for (int j = 0; j < M; j++) {
long long val;
cin >> val;
if (isPerfect(val)) {
ans = max(ans, val);
}
}
}
cout << ans << "\n";
return 0;
}

Độ phức tạp & Cạm bẫy

Thời gian (Time Complexity): O(N × M). Vòng lặp duyệt qua ma trận tốn N × M lần. Tại mỗi phần tử, thao tác so sánh với mảng tốn tối đa 5 bước (hằng số O(1)). Tổng số phép tính khoảng 106, chạy dư sức qua giới hạn 1 giây của hệ thống.

Không gian (Space Complexity): O(1). Chúng ta không cần lưu toàn bộ ma trận vào bộ nhớ mà có thể vừa đọc cin >> val vừa xử lý trực tiếp, tiết kiệm được rất nhiều RAM.

✕ Lỗi dễ mắc: Khởi tạo ans = 0

Nhiều bạn có thói quen khai báo long long ans = 0; khi tìm Max. Tuy nhiên, nếu ma trận không có số hoàn hảo nào, vòng lặp kết thúc và kết quả in ra là 0 thay vì -1 như yêu cầu của đề.

✓ Xử lý đúng: Khởi tạo ans = -1

Bằng cách cho ans = -1 từ đầu. Do mọi số hoàn hảo đều lớn hơn 0, nếu có ít nhất 1 số hoàn hảo tồn tại thì ans chắc chắn sẽ được ghi đè bằng số dương. Nếu không, nó giữ nguyên -1 và là kết quả chính xác để in ra.
