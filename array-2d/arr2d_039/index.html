Tìm khu vực hỏa hoạn lớn nhất

Mô phỏng ngọn lửa lan rộng bằng thuật toán Vết Dầu Loang (Flood Fill)

Cần biết trước: Mảng 2D, Đệ quy hoặc Hàng đợi

Mini-Lab: Chạm vào ô lửa (màu đỏ) để xem cách ngọn lửa lập tức lan ra các ô bên cạnh!

1. Tóm tắt đề bài

Đề bài

Cho ma trận N × M gồm các số 0 (đất trống) và 1 (đang cháy).

Lửa có thể lan sang các ô kề cạnh (trên, dưới, trái, phải). Các ô 1 nối với nhau tạo thành một "đám cháy". Hãy tìm số ô của đám cháy lớn nhất.

Ràng buộc

1 ≤ N, M ≤ 1000

Vì kích thước lớn, thuật toán cần tối ưu O(N·M).

Hướng giải (Logic cốt lõi)

  • Ta muốn gì? Gom tất cả các số 1 nằm cạnh nhau thành từng cụm riêng biệt và đếm kích thước.
  • Quan sát then chốt: Khi đứng ở 1 ô lửa, ta không biết cụm này bự cỡ nào trừ khi "lan" thử ra xung quanh. Nhưng nếu đi không khéo, ta sẽ quay vòng tròn và đếm lặp 1 ô nhiều lần.
  • Cách giải quyết: Quét từ trên xuống dưới. Gặp ô số 1, ta bắt đầu đếm, gọi đệ quy loang ra 4 hướng. Và quan trọng nhất: Đi tới đâu, ta lập tức đổi số 1 thành số 2 (Đã thăm).
  • Vì sao đúng? Việc "phá hủy" ngọn lửa (đổi 1 thành 2) ngay khi đếm xong giúp thuật toán bị "chặn" lại, không bao giờ quay ngược về ô cũ. Mỗi ô lửa chỉ được tính đúng 1 lần cho cụm của nó.

2. Vì sao cần thuật toán Flood Fill?

Thử tưởng tượng nếu ta KHÔNG dùng Flood Fill (Vết Dầu Loang):

Em sẽ phải duyệt từng dòng, cố gắng "nhớ" xem ô 1 ở dòng trên có nối với ô 1 ở dòng dưới hay không. Sẽ ra sao nếu đám cháy có hình chữ U, hoặc hình xoắn ốc? Logic gom nhóm thủ công sẽ trở thành một "ác mộng" với hàng tá biến trạng thái và rất dễ sai.

Flood Fill tận dụng Đệ quy (Recursion) để mô phỏng chính xác hiện tượng tự nhiên: đứng ở tâm, lửa lan ra 4 hướng, các hướng đó lại tiếp tục lan ra tiếp cho đến khi chạm vách hoặc đất trống. Code ngắn gọn, logic tự nhiên, không bao giờ bỏ sót ngóc ngách nào!

3. Mã giả & Cú pháp cốt lõi

Khung thuật toán chuẩn (viết theo phong cách C++) gồm 2 phần:

  1. Hàm đệ quy loang: Nhận tọa độ (r, c), đánh dấu đã thăm, và lan ra 4 hướng.
  2. Vòng lặp chính: Quét cạn bản đồ để tìm những "đốm lửa" chưa bị loang tới.

⏳ Độ phức tạp thời gian: O(N·M)

Mặc dù có đệ quy lồng nhau, nhưng vì lệnh grid[r][c] = 2, mỗi ô trên lưới chỉ được xử lý bên trong hàm đệ quy tối đa 1 lần. Do đó, tổng thời gian tỷ lệ thuận với số lượng ô của ma trận.

int dfs(int r, int c) {
// Dừng nếu ra ngoài mảng hoặc không phải lửa
if (r < 0 || r >= N || c < 0 || c >= M || grid[r][c] != 1) return 0;
grid[r][c] = 2; // Đánh dấu ĐÃ THĂM tránh lặp vô tận
int area = 1; // Tính cả ô hiện tại là 1 đơn vị
// Đệ quy cộng dồn kết quả từ 4 hướng
area += dfs(r - 1, c); // Lên
area += dfs(r + 1, c); // Xuống
area += dfs(r, c - 1); // Trái
area += dfs(r, c + 1); // Phải
return area;
}

Nhấn "Tiến bước" để xem cách máy tính tìm ra đám cháy lớn nhất.
Biến (Main) Biến (Đệ quy) Area Tạm Max Area
i=-, j=- r=-, c=- - 0
int dfs(int r, int c) {
if (r < 0 || r >= N || c < 0 || c >= M || grid[r][c] != 1) return 0;
grid[r][c] = 2;
int area = 1;
area += dfs(r - 1, c);
area += dfs(r + 1, c);
area += dfs(r, c - 1);
area += dfs(r, c + 1);
return area;
}
int maxArea = 0;
for (int i = 0; i < N; i++) {
for (int j = 0; j < M; j++) {
if (grid[i][j] == 1) {
int currentArea = dfs(i, j);
if (currentArea > maxArea) maxArea = currentArea;
}
}
}

5. Cạm bẫy thường gặp (Lỗi kinh điển)

✕ Quên cộng dồn kết quả đệ quy

dfs(r - 1, c);
dfs(r + 1, c);
...
return 1; // SAI

Hậu quả: Thuật toán vẫn loang hết đám cháy, NHƯNG kết quả luôn báo diện tích = 1. Lý do vì bạn "giao việc" cho các nhánh con đi đếm, nhưng lại không thèm thu thập kết quả của chúng để cộng vào biến tổng!

✓ Tích lũy diện tích đúng chuẩn

int area = 1;
area += dfs(r - 1, c);
area += dfs(r + 1, c);
...
return area; // ĐÚNG

Vì sao đúng? Khi nhánh con loang xong và đếm được bao nhiêu ô, nó trả về số lượng đó. Lệnh += giúp gom góp toàn bộ diện tích của 4 hướng lại, cộng với 1 (là chính ô hiện tại) rồi mới gửi lên trên.

6. Kiểm tra nhanh (Quiz)

Câu hỏi: Tại sao dòng lệnh grid[r][c] = 2; lại mang tính sống còn đối với thuật toán này?

Giải thích: Đáp án đúng là B.
Hãy tưởng tượng: Lửa từ ô A lan sang ô B kề bên. Nếu không đánh dấu ô A đã cháy xong, thì từ ô B, thuật toán lại thấy ô A là "lửa (1)" và lan ngược về A. A lại gọi B, B gọi A... Vòng lặp đệ quy này kéo dài vô tận cho đến khi chương trình sập do hết bộ nhớ (Stack Overflow). Việc đổi 1 thành 2 giống như dập tắt lửa ở ô đó sau khi đếm xong!
