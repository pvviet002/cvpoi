1Bài toán mở đầu: hai cách nghĩ cho ra hai kết quả

Cho một xâu gồm chữ cái và chữ số xen kẽ. Hãy tính tổng các số nguyên nằm trong xâu. Ví dụ với xâu abc12de3f45, ba số nằm trong đó là 12, 3 và 45.

Người mới học thường viết ngay: duyệt từng ký tự, gặp ký tự nào là chữ số thì cộng giá trị của nó vào tổng. Một người khác lại gom các chữ số đứng liền nhau thành một số rồi mới cộng. Hai cách cho hai kết quả khác hẳn:

Cách A: cộng từng chữ sốCách B: gom cụm chữ số
Các số được cộng1, 2, 3, 4, 512, 3, 45
Tổng1560

Đề hỏi số nguyên, mà 12 là số mười hai chứ không phải hai số 1 và 2, nên cách B (60) mới đúng. Nhưng vì sao gom cụm lại đúng, và làm thế nào để “biết mình đang ở giữa một cụm số” khi duyệt? Trước khi trả lời, ta cần ba thứ nền tảng:

  • Hiểu kiểu char và sự thật giản dị đứng sau mọi bài xâu: mỗi ký tự là một con số — Mục 2.
  • Biết khai báo và thao tác trên kiểu string — Mục 3.
  • Biết đọc dữ liệu vào cho đúng, chỗ mất điểm oan nhiều nhất — Mục 4.

2Kiểu char: mỗi ký tự là một con số

Một xâu được ghép từ những viên gạch nhỏ nhất là ký tự, và ký tự trong C++ có kiểu riêng: char. Mỗi biến char chứa đúng một ký tự và chiếm 1 byte bộ nhớ.

Điều cốt lõi: char thực chất là một số nguyên

Máy tính không lưu hình dáng chữ cái mà lưu mã ASCII của nó — một số nguyên. Đây là chìa khoá của gần như mọi bài xâu, vì ba dải ký tự thường gặp có mã liên tiếp nhau:

Vì là số nguyên nên char cộng, trừ và so sánh được như số. Từ đó suy ra ba phép quy đổi dùng suốt cả năm học:

  • Vì '0'…'9' liên tiếp nên giá trị của một ký tự chữ số là c - '0' (cho ra 0…9), và “c có phải chữ số” đúng bằng c >= '0' && c <= '9'.
  • Vì 'A' và 'a' cách nhau đúng 32 nên đổi hoa sang thường là c - 'A' + 'a', hoặc gọn hơn là tolower(c).
  • Chữ cái thứ mấy trong bảng chữ: c - 'a' (cho ra 0…25) — chính là chỉ số để đánh dấu vào mảng đếm.

Thư viện cũng có sẵn nhóm hàm phân loại và biến đổi ký tự, đỡ phải nhớ mã:

HàmTrả về đúng khi
isdigit(c)c là chữ số '0'…'9'
isalpha(c)c là chữ cái (hoa hoặc thường)
isalnum(c)c là chữ cái hoặc chữ số — tiện để lọc bỏ dấu câu
isupper(c) / islower(c)c là chữ hoa / chữ thường
isspace(c)c là dấu cách, tab hoặc xuống dòng
toupper(c) / tolower(c)(không phải hàm kiểm tra) đổi c sang chữ hoa / chữ thường

Cả nhóm này nhận vào mã ký tự chứ không nhận char, nên khi truyền hãy ép (unsigned char)c; toupper/tolower trả về int nên gán ngược lại phải ép (char). Chương trình dưới đây gọi thử tất cả:

#include <bits/stdc++.h>
using namespace std;

char c, d;
int i;
string s = "Tin 9!";

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	// char giữ ĐÚNG một ký tự và chiếm 1 byte
	c = 'A';
	cout << "ky tu        : " << c << '\n';
	cout << "kich thuoc   : " << (int) sizeof(char) << " byte\n";
	// Ép sang int để nhìn thấy mã ASCII nằm sau mỗi ký tự
	cout << "ma cua 'A'   : " << (int) 'A' << '\n';
	cout << "ma cua 'a'   : " << (int) 'a' << '\n';
	cout << "ma cua '0'   : " << (int) '0' << '\n';
	// Ký tự thoát: gõ bằng hai ký tự nhưng vẫn chỉ là MỘT ký tự
	cout << "ma cua '\\n'  : " << (int) '\n' << '\n';
	// char là số nguyên nên cộng, trừ, so sánh được như số
	d = c + 1;
	cout << "'A' + 1      : " << d << '\n';
	cout << "'A' < 'a'    : " << ('A' < 'a') << '\n';
	// Ba phép quy đổi dùng suốt khi làm bài
	cout << "gia tri '7'  : " << ('7' - '0') << '\n';
	cout << "'C' ve thuong: " << (char) ('C' - 'A' + 'a') << '\n';
	cout << "'e' thu may  : " << ('e' - 'a' + 1) << '\n';
	// Phân loại từng ký tự của một xâu
	for (i = 0; i < (int) s.size(); i++)
		if (isdigit((unsigned char) s[i])) cout << s[i] << " la chu so\n";
		else if (isupper((unsigned char) s[i])) cout << s[i] << " la chu hoa\n";
		else if (islower((unsigned char) s[i])) cout << s[i] << " la chu thuong\n";
		else if (isspace((unsigned char) s[i])) cout << "_ la khoang trang\n";
		else cout << s[i] << " la ky tu khac\n";
	return 0;
}

Chú ý dòng 'A' + 1 in ra B: phép cộng chạy trên mã số, nhưng vì biến nhận kết quả vẫn là char nên khi in máy lại hiện ký tự. Muốn thấy con số thì phải ép (int).

Vì sao gom cụm mới đúng

Một số nhiều chữ số là giá trị theo cơ số 10: số 12 bằng 1 × 10 + 2, chứ không phải 1 + 2. Khi duyệt, ta dựng dần giá trị ấy bằng

cur = cur × 10 + (c − '0')

Cách A ở Mục 1 sai vì nó cộng tổng các chữ số (1+2=3) thay cho giá trị (12).

Quy tắc chốt cụm. Đang dựng một cụm số mà gặp ký tự không phải chữ số thì cụm đã hết: cộng cur vào kết quả rồi đặt lại cur = 0. Đừng quên cụm số nằm ở cuối xâu — lúc ấy không có ký tự nào “đóng” cụm, nên phải chốt thêm một lần sau vòng lặp.

Cùng một cỗ máy quét — tách từ chính là tách số

Đổi điều kiện “ký tự là chữ số” thành “ký tự khác khoảng trắng” thì đúng khuôn ấy tách được các từ. Trạng thái “đang ở trong một cụm” giữ nguyên vai trò.

Hai cụm “1” liền nhau là hai từ the và cat. Đếm tần suất ký tự cũng là cỗ máy này, chỉ khác trạng thái là một mảng đếm.

Điều đáng nhớ nhất của cả chuyên đề

Xử lý xâu = một vòng for chạy qua các vị trí + số học / trạng thái trên mã ký tự. Gần như mọi bài xâu HSG 9 đều là một biến tấu của câu này.

3Kiểu string và các thao tác

Nhiều ký tự xếp liền nhau thì thành xâu, kiểu string — hiểu đơn giản là một mảng các char đánh chỉ số từ 0, nhưng biết tự co giãn độ dài. Mọi hàm trong mục này đều dùng được trên Themis và có sẵn khi khai báo #include <bits/stdc++.h>.

Bẫy npos — kiểu không dấu

find trả về kiểu không dấu, và npos là giá trị lớn nhất của kiểu ấy chứ không phải −1. Hai cách viết sau đều sai, vì một biểu thức không dấu thì luôn ≥ 0:

if (s.find(x) >= 0)   // luôn đúng, kể cả khi không tìm thấy
if (s.find(x) != -1)  // so sánh lẫn lộn dấu, dễ sinh cảnh báo

Chỉ có một cách viết đúng: if (s.find(x) != string::npos). Cũng vì lý do này mà s.size() - 1 trên một xâu rỗng không cho −1 mà cho một số khổng lồ; khi cần dùng làm chỉ số hãy ép (int) s.size() - 1.

So sánh và thứ tự từ điển

Hai xâu so sánh được trực tiếp bằng ==, !=, <, >. Phép < dùng thứ tự từ điển: dò từ trái sang phải tới vị trí đầu tiên hai xâu khác nhau rồi so mã hai ký tự tại đó. Hai hệ quả hay bị quên:

  • Mã chữ hoa (65…90) đều nhỏ hơn mã chữ thường (97…122), nên "Zoo" < "apple" là đúng. Muốn sắp xếp không phân biệt hoa thường thì phải đưa cả hai về cùng một dạng trước.
  • Nếu xâu này là tiền tố của xâu kia thì xâu ngắn hơn đứng trước: "an" < "anh".

Nhờ vậy sort chạy được thẳng trên vector<string>, và map<string, int> tự giữ khoá theo đúng thứ tự ấy.

Chương trình dưới đây gọi thử lần lượt các hàm ở ba bảng trên:

#include <bits/stdc++.h>
using namespace std;

string s = "Hello, HSG9!";
string t;
vector<string> ds;
size_t vt;
int i;

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	// Kích thước và truy cập từng ký tự (chỉ số bắt đầu từ 0)
	cout << "do dai       : " << s.size() << '\n';
	cout << "ky tu [0]    : " << s[0] << '\n';
	cout << "ky tu cuoi   : " << s.back() << '\n';
	// Cắt xâu con: bắt đầu ở vị trí 7, lấy 4 ký tự
	cout << "substr(7, 4) : " << s.substr(7, 4) << '\n';
	// Bỏ tham số thứ hai thì lấy luôn tới hết xâu
	cout << "substr(7)    : " << s.substr(7) << '\n';
	// Tìm kiếm: trả về vị trí, hoặc string::npos nếu không thấy
	vt = s.find("HSG");
	if (vt != string::npos) cout << "find HSG     : vi tri " << vt << '\n';
	// rfind lấy lần xuất hiện CUỐI cùng; find(x, pos) chỉ tìm từ pos trở đi
	cout << "rfind 'l'    : vi tri " << s.rfind('l') << '\n';
	cout << "find 'H' tu 1: vi tri " << s.find('H', 1) << '\n';
	// Sửa xâu: thay 4 ký tự kể từ vị trí 7 bằng một xâu khác
	t = s;
	t.replace(7, 4, "Tin9");
	cout << "replace      : " << t << '\n';
	// Xoá 2 ký tự kể từ vị trí 5, rồi chèn thêm một xâu vào đầu
	t.erase(5, 2);
	t.insert(0, ">> ");
	cout << "erase+insert : " << t << '\n';
	// Duyệt theo chỉ số như một mảng ký tự, đổi mọi chữ hoa thành chữ thường
	t = "";
	for (i = 0; i < (int) s.size(); i++)
		t += (char) tolower((unsigned char) s[i]);
	cout << "chu thuong   : " << t << '\n';
	// Đảo ngược cả xâu
	reverse(t.begin(), t.end());
	cout << "dao nguoc    : " << t << '\n';
	// Đổi số sang xâu và xâu sang số (dùng stoll cho số vượt int)
	cout << "to_string    : " << to_string(2026) << '\n';
	cout << "stoll + 1    : " << stoll("12345678901") + 1 << '\n';
	// So sánh theo thứ tự từ điển: mọi chữ HOA đều đứng trước chữ thường
	cout << "Zoo < apple  : " << (string("Zoo") < string("apple")) << '\n';
	// Nếu xâu này là tiền tố của xâu kia thì xâu ngắn hơn đứng trước
	cout << "an < anh     : " << (string("an") < string("anh")) << '\n';
	// sort trên vector<string> cũng dùng đúng thứ tự từ điển ấy
	ds.push_back("nam");
	ds.push_back("An");
	ds.push_back("na");
	sort(ds.begin(), ds.end());
	cout << "sort ds      :";
	for (i = 0; i < (int) ds.size(); i++)
		cout << ' ' << ds[i];
	cout << '\n';
	return 0;
}

Thao tác nào nhanh, thao tác nào chậm

Với n là độ dài xâu, m là độ dài mẫu:

Nhanh — O(1)Tốn O(n) mỗi lần gọi
s[i], s.size(), s.back(), s += c, push_back, pop_backinsert, erase, replace, substr, so sánh hai xâu; riêng find mất tới O(n × m)

Hai lỗi làm chương trình đúng vẫn bị quá thời gian:

  • s = s + c dựng hẳn một xâu mới mỗi vòng lặp, nên nối n ký tự mất O(n2). Luôn viết s += c.
  • Gọi s.erase(i, 1) trong vòng lặp để lọc ký tự cũng là O(n2). Thay bằng cách dựng một xâu kết quả mới và chỉ += những ký tự được giữ.

  • string_view (C++17) — bộ dịch báo lỗi was not declared. Cứ dùng string thường.
  • stol("3000000000") — long chỉ 4 byte nên số vượt 2,1 tỉ làm hàm ném lỗi out_of_range. Với số lớn phải dùng stoll (trả long long 8 byte).
  • __int128 — bị từ chối hẳn. Số quá long long thì phải xử lý dưới dạng xâu (Mục 7).

Đây là chỗ mất điểm oan nhiều nhất: thuật toán đúng nhưng đọc sai dữ liệu thì sai từ test đầu tiên.

Đọc một từ hay đọc cả dòng

cin >> s đọc một “từ”: nó bỏ qua khoảng trắng đầu rồi đọc tới khi gặp khoảng trắng kế tiếp — nên không đọc được tên có dấu cách như “Nguyen Van An”. Muốn lấy cả dòng thì dùng getline(cin, s). Cái bẫy hay gặp: sau khi cin >> n đọc một số, ký tự xuống dòng vẫn còn nằm lại trong bộ đệm, khiến getline ngay sau đó đọc phải một dòng rỗng. Phải gọi cin.ignore() để bỏ ký tự thừa ấy.

#include <bits/stdc++.h>
using namespace std;

int n;
string ten;

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	// Đọc một số nguyên, rồi đọc cả một dòng họ tên (có dấu cách)
	cin >> n;
	cin.ignore();   // bỏ ký tự xuống dòng còn sót lại sau khi đọc số
	getline(cin, ten);
	cout << "So the: " << n << '\n';
	cout << "Ho ten: " << ten << " (" << ten.size() << " ky tu)\n";
	return 0;
}
Chạy thử

Với dữ liệu vào gồm dòng 7 rồi dòng Nguyen Van An, chương trình in:

So the: 7
Ho ten: Nguyen Van An (13 ky tu)

Nếu quên cin.ignore(), getline sẽ đọc phải dòng rỗng và họ tên bị mất.

Tách một dòng thành nhiều phần

Đọc được cả dòng rồi thì thường phải tách nó ra. Có hai công cụ, chọn theo dấu phân cách:

  • Phân cách là khoảng trắng → đổ dòng vào một istringstream rồi dùng >> như đọc từ bàn phím. Cách này tự bỏ qua mọi dấu cách thừa, kể cả nhiều dấu cách liền nhau.
  • Phân cách là ký tự khác (dấu phẩy, dấu chấm phẩy…) → dùng getline(luong, phan, ','). Khác với >>, cách này giữ lại cả trường rỗng nằm giữa hai dấu phẩy liền nhau.
#include <bits/stdc++.h>
using namespace std;

string dong, tu, phan;
map<string, int> dem;
map<string, int>::iterator it;
int soTu;

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	getline(cin, dong);
	// stringstream biến một dòng thành luồng, rồi >> lấy ra từng từ;
	// nhiều dấu cách liền nhau cũng được bỏ qua tự động
	istringstream luong(dong);
	while (luong >> tu) {
		soTu++;
		dem[tu]++;
	}
	cout << "So tu: " << soTu << '\n';
	// map tự sắp xếp khoá theo thứ tự từ điển nên không cần sort lại
	for (it = dem.begin(); it != dem.end(); it++)
		cout << it->first << ' ' << it->second << '\n';
	// getline có tham số thứ ba: cắt theo một dấu phân cách bất kỳ.
	// Khác >> ở chỗ nó GIỮ lại cả trường rỗng giữa hai dấu phẩy liền nhau
	getline(cin, dong);
	istringstream luong2(dong);
	while (getline(luong2, phan, ','))
		cout << '[' << phan << ']';
	cout << '\n';
	return 0;
}

5Khuôn mẫu: quét cụm số

Ghép ba điều ở Mục 2 — kiểm tra chữ số, dựng giá trị cơ số 10, và chốt cụm — ta được khuôn giải trọn bài toán mở đầu. Lưu ý dùng long long cho cur và tong vì tổng có thể vượt int.

#include <bits/stdc++.h>
using namespace std;

string s;
long long cur, tong;
int i;
bool trongCum;

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	getline(cin, s);
	// Quét từng ký tự, gom các chữ số đứng liền nhau thành MỘT số
	for (i = 0; i < (int) s.size(); i++)
		if (s[i] >= '0' && s[i] <= '9') {
			cur = cur * 10 + (s[i] - '0');   // xây số theo cơ số 10
			trongCum = true;
		} else if (trongCum) {
			tong += cur;   // chốt cụm số vừa kết thúc
			cur = 0;
			trongCum = false;
		}
	if (trongCum) tong += cur;   // cụm số nằm ở cuối xâu
	cout << tong << '\n';
	return 0;
}

6Bốn khái niệm phải phân biệt

Đề bài hay dùng bốn từ nghe na ná nhau nhưng chỉ những tập hoàn toàn khác nhau. Nhầm một chữ là làm sai cả bài, nên bốn định nghĩa dưới đây phải thuộc. Lấy xâu abcd làm ví dụ:

Tiền tố và hậu tố đều là trường hợp riêng của xâu con; xâu con lại là trường hợp riêng của dãy con. Con số ở cột cuối giải thích vì sao đề về dãy con gần như luôn cần quy hoạch động: với n = 30 đã có hơn một tỉ dãy con, không thể duyệt hết.

#include <bits/stdc++.h>
using namespace std;

string s = "abc";
int i, j, mask, n;

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	n = (int) s.size();
	// Tiền tố: luôn bắt đầu từ vị trí 0, độ dài chạy từ 1 tới n
	cout << "Tien to :";
	for (i = 1; i <= n; i++)
		cout << ' ' << s.substr(0, i);
	cout << '\n';
	// Hậu tố: luôn kéo tới hết xâu, điểm bắt đầu chạy từ 0 tới n-1
	cout << "Hau to  :";
	for (i = 0; i < n; i++)
		cout << ' ' << s.substr(i);
	cout << '\n';
	// Xâu con: các ký tự phải LIỀN NHAU — chọn điểm đầu i và độ dài j
	cout << "Xau con :";
	for (i = 0; i < n; i++)
		for (j = 1; i + j <= n; j++)
			cout << ' ' << s.substr(i, j);
	cout << '\n';
	// Dãy con: KHÔNG cần liền nhau — mỗi ký tự chỉ có hai lựa chọn lấy hoặc bỏ,
	// nên duyệt bằng cách cho mask chạy qua mọi tổ hợp bit
	cout << "Day con :";
	for (mask = 1; mask < (1 << n); mask++) {
		cout << ' ';
		for (i = 0; i < n; i++)
			if (mask & (1 << i)) cout << s[i];
	}
	cout << '\n';
	cout << "So luong: " << n * (n + 1) / 2 << " xau con, "
	     << (1 << n) - 1 << " day con (deu khong tinh xau rong)\n";
	return 0;
}

Xâu đối xứng và kỹ thuật hai con trỏ

Một xâu đối xứng (palindrome) là xâu đọc xuôi bằng đọc ngược, tức là s[i] luôn bằng s[n-1-i]. Cách kiểm chuẩn là hai con trỏ: đặt một biến ở đầu, một biến ở cuối, so hai ký tự rồi cho hai biến tiến vào giữa. Chỉ cần một lần lệch là kết luận ngay, không phải duyệt nốt.

#include <bits/stdc++.h>
using namespace std;

string s, t;
int trai, phai;
bool doiXung;

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	getline(cin, s);
	// Hai con trỏ: một chạy từ đầu, một chạy từ cuối, tiến về giữa
	trai = 0;
	phai = (int) s.size() - 1;
	doiXung = true;
	while (trai < phai) {
		if (s[trai] != s[phai]) {
			doiXung = false;
			break;
		}
		trai++;
		phai--;
	}
	cout << "Hai con tro : " << (doiXung ? "YES" : "NO") << '\n';
	// Cách thứ hai, ngắn hơn nhưng tốn thêm một xâu nhớ bản đảo ngược
	t = s;
	reverse(t.begin(), t.end());
	cout << "So voi dao  : " << (s == t ? "YES" : "NO") << '\n';
	return 0;
}

Mảng đếm ký tự

Khuôn nhỏ đáng thuộc lòng: mảng cnt kích thước 256 (đủ cho mọi mã ký tự 1 byte), lấy mã ký tự làm chỉ số. Ví dụ dưới gộp chữ hoa vào chữ thường rồi tìm chữ cái xuất hiện nhiều nhất.

#include <bits/stdc++.h>
using namespace std;

string s;
int cnt[256];
int i, best;
char c, kytu;

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	getline(cin, s);
	// Đếm tần suất: mã ký tự làm chỉ số mảng, ép unsigned để không bị âm
	for (i = 0; i < (int) s.size(); i++) {
		c = s[i];
		if (c >= 'A' && c <= 'Z') c = c - 'A' + 'a';   // gộp hoa vào thường
		cnt[(unsigned char) c]++;
	}
	// Tìm chữ cái thường xuất hiện nhiều nhất (hoà thì lấy chữ đứng trước)
	best = -1;
	kytu = '?';
	for (c = 'a'; c <= 'z'; c++)
		if (cnt[(unsigned char) c] > best) {
			best = cnt[(unsigned char) c];
			kytu = c;
		}
	cout << kytu << ' ' << best << '\n';
	return 0;
}

Cùng mảng đếm ấy, thêm một dạng bài

Đếm số lần một mẫu xuất hiện

find chỉ cho vị trí đầu tiên; muốn đếm hết thì gọi lại nhiều lần với tham số pos. Điểm phải quyết định trước khi viết: bước nhảy sau mỗi lần khớp.

#include <bits/stdc++.h>
using namespace std;

string s, mau;
size_t vt;
int demChong, demRoi;

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	getline(cin, s);
	getline(cin, mau);
	if (mau.empty()) return 0;
	// Đếm CÓ chồng lấn: tìm thấy rồi chỉ nhích sang phải một bước
	vt = s.find(mau);
	while (vt != string::npos) {
		demChong++;
		vt = s.find(mau, vt + 1);
	}
	// Đếm KHÔNG chồng lấn: nhảy hẳn qua độ dài của mẫu vừa khớp
	vt = s.find(mau);
	while (vt != string::npos) {
		demRoi++;
		vt = s.find(mau, vt + mau.size());
	}
	cout << "Co chong lan   : " << demChong << '\n';
	cout << "Khong chong lan: " << demRoi << '\n';
	return 0;
}

Ba vị trí khớp là 0, 1, 2 nếu cho phép chồng lấn; chỉ còn 0 và 2 nếu mỗi ký tự chỉ được dùng một lần. Đề thường không nói rõ, nên phải nhìn ví dụ mẫu mà chọn. Cách đếm này mất O(n × m) — với n, m cỡ vài nghìn thì vẫn thừa nhanh trong 1 giây.

Chuẩn hoá xâu

Dạng “bỏ khoảng trắng thừa”, “viết hoa chữ cái đầu mỗi từ” xuất hiện rất đều. Mẹo là đừng sửa tại chỗ trên xâu gốc — cứ tách từ rồi dựng một xâu kết quả mới, vừa ngắn vừa tránh lỗi thừa dấu cách ở đầu và cuối.

#include <bits/stdc++.h>
using namespace std;

string dong, tu, kq;
int i;

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	getline(cin, dong);
	// Đọc bằng luồng thì mọi khoảng trắng thừa ở đầu, giữa và cuối tự mất
	istringstream luong(dong);
	while (luong >> tu) {
		// Chữ cái đầu từ viết hoa, các chữ còn lại đưa hết về chữ thường
		for (i = 0; i < (int) tu.size(); i++)
			if (i == 0) tu[i] = (char) toupper((unsigned char) tu[i]);
			else tu[i] = (char) tolower((unsigned char) tu[i]);
		// Chỉ chèn dấu cách TRƯỚC từ thứ hai trở đi, tránh thừa cách ở cuối
		if (!kq.empty()) kq += ' ';
		kq += tu;
	}
	cout << kq << '\n';
	return 0;
}

Dòng vào " nguyen VAN an " (thừa dấu cách ở đầu, ở cuối và giữa các từ) cho ra đúng Nguyen Van An.

Chi tiết đáng để ý là if (!kq.empty()) kq += ' '; — chèn dấu cách trước mỗi từ kể từ từ thứ hai, thay vì chèn sau mỗi từ rồi phải xoá dấu cách thừa ở cuối.

Số lớn hơn long long

Khi nào phải dùng xâu để chứa số

long long chứa được số tới khoảng 9,2 × 1018, tức 19 chữ số. Đề nào cho số tới hàng trăm chữ số thì không kiểu dựng sẵn nào chứa nổi, mà __int128 lại không dùng được trên Themis (Mục 3). Lối ra duy nhất: để nguyên số dưới dạng xâu và cộng trừ theo cột, đúng như cách đặt tính dọc đã học ở tiểu học.

Ba điểm phải nhớ khi cài: cộng bắt đầu từ cuối xâu (hàng đơn vị); vòng lặp chỉ dừng khi cả hai số đã hết và không còn nhớ; và kết quả dựng ra đang ngược nên phải reverse trước khi in.

#include <bits/stdc++.h>
using namespace std;

string a, b, kq;
int i, j, nho, tong;

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	cin >> a >> b;
	// Cộng bắt đầu từ hàng đơn vị, tức là từ CUỐI xâu ngược về đầu
	i = (int) a.size() - 1;
	j = (int) b.size() - 1;
	// Còn chữ số ở một trong hai số, hoặc còn nhớ, thì còn phải cộng tiếp
	while (i >= 0 || j >= 0 || nho > 0) {
		tong = nho;
		if (i >= 0) tong += a[i--] - '0';
		if (j >= 0) tong += b[j--] - '0';
		kq += (char) (tong % 10 + '0');   // chữ số của hàng đang xét
		nho = tong / 10;                  // phần nhớ đẩy sang hàng kế tiếp
	}
	// Các chữ số đang nằm ngược, đảo lại mới đúng thứ tự đọc
	reverse(kq.begin(), kq.end());
	cout << kq << '\n';
	return 0;
}

Điều kiện i >= 0 || j >= 0 || nho > 0 lo trọn ba việc một lúc: hai số dài ngắn khác nhau, và phần nhớ cuối cùng làm kết quả dài thêm một chữ số. Bỏ sót vế nho > 0 là mất đúng chữ số 1 đứng đầu.

  • “các từ cách nhau bởi dấu cách”, “đếm số từ” ⇒ istringstream (Mục 4).

  1. cin >> s chỉ đọc một từ. Tên “Nguyen Van An” bị mất phần sau dấu cách. Cần cả dòng thì dùng getline.
  2. Trộn cin >> n rồi getline thì phải cin.ignore(), nếu không getline đọc phải dòng rỗng.
  3. So với string::npos, đừng so với −1 hay >= 0 — find trả kiểu không dấu nên hai cách kia luôn cho kết quả đúng, che mất trường hợp không tìm thấy.
  4. s.size() cũng không dấu. Viết (int) s.size() - 1 khi dùng làm chỉ số, kẻo xâu rỗng cho ra một số khổng lồ.
  5. char có thể mang mã âm với ký tự ngoài bảng ASCII chuẩn. Khi dùng làm chỉ số mảng đếm phải ép (unsigned char)c, kẻo tràn ra ngoài mảng.
  6. long chỉ 4 byte trên Themis — tổng lớn phải khai long long, và đổi xâu sang số lớn phải dùng stoll chứ không phải stol.
  7. Chỉ số đếm từ 0: ký tự cuối là s[s.size()-1], không phải s[s.size()].
  8. Cụm số cuối xâu dễ bị quên chốt — luôn xử lý thêm một lần sau vòng lặp.
  9. Viết s += c, đừng viết s = s + c; và đừng gọi erase trong vòng lặp — cả hai biến bài O(n) thành O(n2).
  10. Xâu rỗng là trường hợp hợp lệ. Dòng dữ liệu có thể trống; kiểm xem chương trình có chạy đúng với s.size() == 0 không.
