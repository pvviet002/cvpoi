Chi Phí Chuyển Tuyến

Bản chất của bài toán này không nằm ở khoảng cách vật lý giữa các trạm, mà nằm ở chi phí lên xe (chuyển tuyến). Bằng cách phân rã mỗi trạm thành các trạng thái "đang ngồi trên tuyến cụ thể", ta đưa bài toán về tìm đường đi ngắn nhất thông thường.

Mini-lab: Chi phí đi từ A đến B

Trạm A (Đang đi bộ)
Trạm B

Tóm tắt đề bài

Có n trạm và m đoạn đường. Mỗi đoạn đường thuộc đúng một trong k tuyến xe. Tuyến c có giá vé là pc.

Hành khách chỉ bị trừ tiền (bằng pc) khi bắt đầu bước lên tuyến c. Nếu đi liên tiếp nhiều đoạn trên CÙNG tuyến c thì không mất thêm phí. Nếu đổi sang tuyến khác rồi quay lại tuyến c, phải trả lại pc.

Yêu cầu: Tìm chi phí nhỏ nhất đi từ trạm 1 đến trạm n.

Ràng buộc:
1 ≤ n, k ≤ 200000
0 ≤ m ≤ 300000
1 ≤ pc ≤ 109

Ví dụ Input:
6 6 3 (6 trạm, 6 đường, 3 tuyến)
5 2 9 (giá tuyến 1, 2, 3)
1 2 1 (từ 1-2 tuyến 1)
2 3 1 (từ 2-3 tuyến 1)
...

Phân tích ban đầu: Tại sao Dijkstra thông thường thất bại?

Nếu ta chạy thuật toán Dijkstra trên đồ thị n đỉnh (mỗi đỉnh là 1 trạm), trọng số cạnh là gì? Nếu gán cạnh (u, v) trên tuyến c có trọng số pc, ta sẽ cộng dồn sai chi phí khi đi liên tiếp trên cùng một tuyến. Nếu gán trọng số bằng 0, ta lại không tính được chi phí lúc mới lên xe.

Khó khăn then chốt: Chi phí đi qua một cạnh (u, v) thuộc tuyến c phụ thuộc vào cạnh trước đó bạn vừa đi thuộc tuyến nào. Đồ thị thiếu "ký ức".

Mô-đun 1: Tách Trạm thành Trạng thái (Đồ thị Ký ức)

(a) Ta muốn gì? Ta muốn đồ thị có khả năng "nhớ" được hành khách đang ngồi trên tuyến xe nào, để quyết định có thu tiền hay không.

(b) Khó khăn: Một trạm u có thể là điểm giao của rất nhiều tuyến xe.

(c) Ý tưởng vượt qua: Thay vì chỉ có đỉnh u đại diện cho trạm, ta tạo ra các đỉnh ảo dạng (u, c), mang ý nghĩa: "Đang ở trạm u, và ĐANG NGỒI trên xe của tuyến c".
Ngoài ra, ta giữ lại đỉnh u nguyên bản mang ý nghĩa "Đang đứng dưới đất tại trạm u".

(d) Vì sao đúng? Khi tách như vậy, quá trình đi lại được phân tách rõ ràng:
- Từ u (đứng dưới đất) bước lên (u, c) (lên xe): Trả phí pc.
- Từ (u, c) đi tới (v, c) (xe di chuyển): Trả phí 0.
- Từ (v, c) xuống v (xuống đất): Trả phí 0.

Mô phỏng 1: Các thao tác chuyển trạng thái tại trạm 3

Đang đứng dưới đất tại Trạm 3. Chi phí hiện tại: 0. Hãy thử lên xe.

Câu hỏi kiểm tra

Giả sử bạn đang ở trạng thái (u, Tuyến 1). Bạn muốn đi tiếp sang một đỉnh v thông qua cạnh thuộc Tuyến 2. Chuỗi thao tác đúng (và rẻ nhất) trên đồ thị mới là gì?

Đúng! Đồ thị của chúng ta KHÔNG có cạnh nối trực tiếp giữa hai tuyến xe khác nhau. Bắt buộc phải thực hiện thao tác "xuống xe" (quay về đỉnh u gốc) rồi mới "lên xe" (chuyển sang (u, Tuyến 2)). Điều này mô phỏng hoàn hảo việc đổi tuyến và tự động tính thêm phí p2.

Mô-đun 2: Tối ưu Không Gian (Đồ thị phân tầng)

(a) Ta muốn gì? Ta muốn biểu diễn trạng thái (u, c) vào một cấu trúc dữ liệu để chạy Dijkstra.

(b) Khó khăn: Nếu tạo trạng thái cho MỌI trạm kết hợp với MỌI tuyến, số đỉnh sẽ là N × K (có thể lên tới 200,000 × 200,000 = 40 tỉ đỉnh). Bị Tràn Bộ Nhớ (MLE).

(c) Ý tưởng vượt qua: Ta chỉ tạo trạng thái (u, c) NẾU có ít nhất một đoạn đường của tuyến c nối trực tiếp vào trạm u. Thay vì mảng 2 chiều, ta gán cho mỗi cặp (u, c) hợp lệ một số ID duy nhất bằng mảng 1 chiều liên tiếp.

(d) Vì sao đúng? Mỗi cạnh của đồ thị gốc đóng góp tối đa 2 trạng thái hợp lệ. Tổng số đỉnh (u, c) tối đa chỉ là 2 × M. Cộng với N đỉnh trạm gốc, tổng số đỉnh trong đồ thị mới không vượt quá N + 2M ≤ 800,000. Dư sức chạy Dijkstra!

Câu hỏi kiểm tra

Trong đồ thị phân tầng mới, có bao nhiêu loại cạnh, và trọng số của chúng lần lượt là bao nhiêu?

Chính xác! Bí quyết là bạn chỉ trả phí MỘT LẦN khi bắt đầu lên xe (tức là cạnh đi từ dưới đất lên tầng tuyến xe). Mọi cạnh di chuyển ngang trên cùng một tầng, hoặc cạnh nhảy xuống đất đều MIỄN PHÍ.

Thuật toán từng bước

  1. Lưu các tuyến đi qua mỗi trạm:
    • Tạo mảng 2 chiều động vector<int> lines_at[n + 1].
    • Với mỗi cạnh (u, v) thuộc tuyến c, đẩy c vào lines_at[u] và lines_at[v].
    • Loại bỏ phần tử trùng trong mỗi lines_at[u] bằng cách sort() rồi dùng unique().
  2. Định danh (ID) cho đỉnh ảo:
    • Các đỉnh dưới đất có ID từ 1 đến n.
    • Biến đếm tổng số đỉnh đồ thị mới: V = n.
    • Tạo mảng start_id[u] để biết ID ảo của trạm u bắt đầu từ đâu. Với mỗi trạm u, gán start_id[u] = V, sau đó V = V + lines_at[u].size().
    • Hàm lấy ID của trạng thái (u, c): tìm vị trí của c trong lines_at[u] (gọi là idx), ID trả về là start_id[u] + idx + 1.
  3. Xây dựng tập cạnh đồ thị mới:
    • Tạo danh sách kề adj[V + 1].
    • Với mỗi trạm u và mỗi tuyến c trong lines_at[u]:
      • Gọi id = get_id(u, c).
      • Thêm cạnh: u → id trọng số pc (Lên xe).
      • Thêm cạnh: id → u trọng số 0 (Xuống xe).
    • Với mỗi cạnh ban đầu (u, v, c):
      • Gọi id_u = get_id(u, c) và id_v = get_id(v, c).
      • Thêm cạnh vô hướng: id_u ↔ id_v trọng số 0 (Di chuyển).
  4. Chạy Dijkstra:
    • Khởi tạo mảng dist[V + 1] vô cùng lớn, dist[1] = 0.
    • Dùng hàng đợi ưu tiên priority_queue lưu cặp (khoảng cách, đỉnh).
    • Chạy cho đến khi lấy ra đỉnh n (vì yêu cầu đến trạm n dưới đất). Đáp số chính là dist[n]. Nếu hàng đợi rỗng mà chưa tới n, in ra -1.

Cài đặt (C++)

#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>
using namespace std;
const long long INF = 1e18;
struct Edge { int to, weight; };
struct InputEdge { int u, v, c; };
int main() {
ios_base::sync_with_stdio(false); cin.tie(NULL);
int n, m, k;
if (!(cin >> n >> m >> k)) return 0;
vector<int> p(k + 1);
for (int i = 1; i <= k; i++) cin >> p[i];
vector<InputEdge> edges(m);
vector<vector<int>> lines_at(n + 1);
for (int i = 0; i < m; i++) {
cin >> edges[i].u >> edges[i].v >> edges[i].c;
lines_at[edges[i].u].push_back(edges[i].c);
lines_at[edges[i].v].push_back(edges[i].c);
}
// Bước 1 & 2: Lọc tuyến độc nhất và thiết lập hệ thống định danh ID
int V = n;
vector<int> start_id(n + 1, 0);
for (int i = 1; i <= n; i++) {
sort(lines_at[i].begin(), lines_at[i].end());
lines_at[i].erase(unique(lines_at[i].begin(), lines_at[i].end()), lines_at[i].end());
start_id[i] = V;
V += lines_at[i].size();
}
// Hàm helper lấy ID ảo của trạng thái (trạm u, tuyến c)
auto get_id = [&](int u, int c) {
auto it = lower_bound(lines_at[u].begin(), lines_at[u].end(), c);
int idx = distance(lines_at[u].begin(), it);
return start_id[u] + idx + 1;
};
// Bước 3: Xây dựng đồ thị mới
vector<vector<Edge>> adj(V + 1);
for (int i = 1; i <= n; i++) {
for (int c : lines_at[i]) {
int id = get_id(i, c);
adj[i].push_back({id, p[c]}); // Đi lên xe tốn phí
adj[id].push_back({i, 0}); // Đi xuống đất miễn phí
}
}
for (auto& e : edges) {
int id_u = get_id(e.u, e.c);
int id_v = get_id(e.v, e.c);
adj[id_u].push_back({id_v, 0}); // Xe chạy giữa 2 trạm miễn phí
adj[id_v].push_back({id_u, 0});
}
// Bước 4: Dijkstra tìm đường ngắn nhất
vector<long long> dist(V + 1, INF);
using pqi = pair<long long, int>;
priority_queue<pqi, vector<pqi>, greater<pqi>> pq;
dist[1] = 0;
pq.push({0, 1});
while (!pq.empty()) {
auto [d, u] = pq.top(); pq.pop();
if (d > dist[u]) continue;
if (u == n) { // Tới đích
cout << d << "\n";
return 0;
}
for (auto& edge : adj[u]) {
if (dist[edge.to] > dist[u] + edge.weight) {
dist[edge.to] = dist[u] + edge.weight;
pq.push({dist[edge.to], edge.to});
}
}
}
cout << (dist[n] == INF ? -1 : dist[n]) << "\n";
return 0;
}

Đánh giá & Cạm bẫy

Độ phức tạp:
- Khởi tạo ID mảng: Việc sắp xếp tập tuyến cho mỗi đỉnh mất O(M log(K)).
- Số đỉnh đồ thị V ≤ N + 2M ≤ 800,000. Số cạnh E ≤ 6M ≤ 1,800,000.
- Chạy Dijkstra tốn O(E log V). Tổng cộng thuật toán chạy cực nhanh, hoàn toàn vừa vặn với giới hạn thời gian 1-2 giây cho mảng ràng buộc 105.

Cạm bẫy dễ sai

Sử dụng std::map<pair<int, int>, int> để đánh số ID cho trạng thái. Dù tiện lợi nhưng map sử dụng cây đỏ đen, tốn rất nhiều bộ nhớ và hằng số thời gian cực lớn (gấp 5-10 lần mảng). Dễ bị Tràn Thời Gian hoặc Tràn Bộ Nhớ với M = 300,000.

Cách xử lý đúng

Thu gom tất cả các tuyến kề với từng trạm u vào một vector, gọi sort() và unique() để loại trùng lặp. Gán ID theo đoạn liên tiếp bằng tổng tiền tố. Tìm kiếm ID ảo bằng thuật toán chặt nhị phân lower_bound rất nhẹ và không tốn thêm bộ nhớ dư.
