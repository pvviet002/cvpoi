Bấm vào các thẻ để đổi chỗ. Bạn thấy cách ghép nào tạo ra số lớn hơn?

1.Tóm tắt đề bài

Cho N tấm thẻ chứa các dãy chữ số. Cần xếp chúng thành hàng ngang để khi đọc ghép lại, ta được số nguyên lớn nhất.

  • Input: N tấm thẻ (1 ≤ N ≤ 105). Tổng độ dài các dãy chữ số ≤ 106. Các dãy không có số 0 đứng đầu (trừ khi dãy là "0").
  • Output: Dãy chữ số ghép được lớn nhất, không có số 0 vô nghĩa đứng đầu.
Input: 3 30 34 5 9
Output: 9534330

2.Phân tích ban đầu

Nếu N rất nhỏ (N ≤ 8), ta có thể sinh tất cả N! hoán vị và so sánh để chọn kết quả tốt nhất. Tuy nhiên, với N lên đến 105, cách này bất khả thi.

Chúng ta muốn số được tạo ra có các chữ số đầu tiên càng lớn càng tốt. Điều này gợi ý ta nên sắp xếp các thẻ ưu tiên những chữ số lớn lên đầu (ví dụ: "9" phải đứng trước "5"). Nhưng liệu sắp xếp từ điển (lexicographical sort) thông thường có đủ tốt?

3.1. Cạm bẫy của sắp xếp từ điển

Mục tiêu: Đưa các chuỗi "lớn" lên trước.

Khó khăn: Sắp xếp từ điển mặc định so sánh từng ký tự từ trái qua phải. Nếu một chuỗi là tiền tố của chuỗi kia, chuỗi dài hơn sẽ được coi là lớn hơn. Ví dụ, hệ thống đánh giá "30" > "3".

Nếu ta xếp theo thứ tự giảm dần mặc định, "30" sẽ đứng trước "3", tạo ra số 303. Tuy nhiên, nếu xếp "3" trước "30", ta được số 330. Rõ ràng 330 > 303.

Hãy xem khác biệt với thẻ 34 và 3:

34 + 3 = 343 > 3 + 34 = 334

Ở đây "34" đứng trước lại cho kết quả đúng, khác với "30" và "3". Vậy ta không thể chỉ dựa vào độ dài hay từ điển thuần túy.

Câu hỏi kiểm tra

Nếu bạn có hai thẻ "12" và "121", thứ tự nào sẽ tạo ra số ghép lớn hơn?

A. Xếp "121" trước, thành 12112
B. Xếp "12" trước, thành 12121
Giải thích: Mặc dù "121" > "12" theo từ điển, nhưng khi ghép lại: "12" + "121" = 12121, lớn hơn "121" + "12" = 12112. Do đó phải xếp "12" trước.

4.2. Quy tắc ghép đôi (Custom Comparator)

Ý tưởng: Thay vì cố gắng tìm các luật phức tạp về tiền tố hay độ dài, ta chỉ cần đặt câu hỏi trực tiếp: Giữa thẻ A và thẻ B, ai đứng trước thì có lợi hơn?

Cách kiểm tra nhanh nhất là thử ghép cả hai trường hợp: ghép A rồi B (ký hiệu A + B), và ghép B rồi A (ký hiệu B + A). Nếu chuỗi (A + B) lớn hơn chuỗi (B + A), thì chắc chắn A phải đứng trước B trong mảng.

Vì sao nó đúng? Quan hệ A + B > B + A có tính chất bắc cầu. Nghĩa là nếu A nên đứng trước B, và B nên đứng trước C, thì tự động A sẽ hợp lý khi đứng trước C. Ta có thể dùng quy tắc này làm hàm so sánh (comparator) cho các thuật toán sắp xếp chuẩn.

vs
934 > 349
Nên xếp 9 đứng trước 34

Câu hỏi kiểm tra

Trong C++, hàm std::sort cần một hàm so sánh trả về true nếu phần tử a phải đứng trước phần tử b. Với logic trên, hàm so sánh sẽ viết như thế nào?

A. return a > b;
B. return a + b > b + a;
C. return a.length() > b.length();
Giải thích: Hàm return a + b > b + a; chính là cốt lõi của bài toán. Nó tạo chuỗi mới và so sánh trực tiếp hiệu quả của việc xếp a trước b.

5.3. Chạy thuật toán trên mảng

Bây giờ ta áp dụng quy tắc A + B > B + A để sắp xếp toàn bộ mảng. Dưới đây là mô phỏng quá trình đưa các thẻ lớn nhất lên đầu (tương tự thuật toán sắp xếp nổi bọt để dễ quan sát từng cặp so sánh).

bool compareCards(const string& a, const string& b) { return a + b > b + a; } // Thuật toán duyệt các cặp và đổi chỗ nếu cần
Bấm Tiến để bắt đầu mô phỏng.
0/0

Thuật toán từng bước

  1. Đọc dữ liệu:
    • Khởi tạo một mảng chuỗi a để lưu N dãy chữ số.
    • Đọc từng chuỗi từ input vào a[i].
  2. Sắp xếp mảng:
    • Sử dụng hàm sắp xếp có sẵn (như std::sort).
    • Truyền vào hàm so sánh tùy chỉnh: với hai chuỗi x và y, trả về true nếu x + y > y + x.
  3. Xử lý trường hợp đặc biệt (Zero case):
    • Nếu sau khi sắp xếp, phần tử đầu tiên của mảng a[0] == "0", điều này có nghĩa là tất cả các thẻ đều là "0".
    • Ta chỉ in ra một số 0 duy nhất và kết thúc chương trình, tránh việc in ra "000...0".
  4. In kết quả:
    • Nếu không rơi vào trường hợp trên, duyệt mảng a từ đầu đến cuối.
    • In liên tiếp các chuỗi a[i] không có khoảng trắng.

6.Cài đặt (C++)

#include #include #include #include using namespace std; // Hàm so sánh tùy chỉnh cho std::sort bool compareCards(const string& a, const string& b) { return a + b > b + a; } int main() { // Tối ưu I/O cho C++ ios_base::sync_with_stdio(false); cin.tie(NULL); int n; if (!(cin >> n)) return 0; vector a(n); for (int i = 0; i < n; i++) { cin >> a[i]; } // Sắp xếp mảng bằng hàm compareCards sort(a.begin(), a.end(), compareCards); // Cạm bẫy: Toàn số 0 if (a[0] == "0") { cout << "0\n"; } else { // In kết quả for (int i = 0; i < n; i++) { cout << a[i]; } cout << "\n"; } return 0; }

7.Độ phức tạp

  • Thời gian: Thuật toán std::sort thực hiện khoảng O(N log N) phép so sánh. Mỗi phép so sánh thực hiện nối hai chuỗi có tổng độ dài trung bình là L, với thao tác so sánh chuỗi tốn O(L). Tổng thời gian xấp xỉ O(L · N log N). Tuy nhiên, tổng độ dài mọi chuỗi ≤ 106, nên thực tế việc nối và so sánh diễn ra rất nhanh. Hoàn toàn vượt qua giới hạn 1 giây của ràng buộc.
  • Không gian (Bộ nhớ): Ta lưu trữ mảng N phần tử và tổng các ký tự của chuỗi mất O(Tổng độ dài chuỗi) ≤ 106 bytes (khoảng vài MB). Phù hợp với mọi giới hạn bộ nhớ chuẩn.

8.Cạm bẫy khi cài đặt

✕ Bỏ qua mảng chỉ chứa số 0

Nếu input là 3\n0 0 0, in thẳng mảng sau sắp xếp sẽ ra 000. Đề bài cấm số 0 vô nghĩa đứng đầu.

✓ Kiểm tra phần tử đầu tiên

Mảng đã xếp giảm dần. Nếu thẻ lớn nhất nằm ở đầu tiên a[0] == "0", nghĩa là mọi thẻ khác cũng chỉ là "0". Ta bắt nhánh này và chỉ in "0".
